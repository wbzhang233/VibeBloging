## 核心认知框架

**Claude Code 不是"AI 聊天工具"，而是一台精密的 harness engineering 机器。**
- **Harness（驱动层）**：为 LLM 提供可操作环境，让推理结果能影响真实世界
- **Model（推理核心）**：Claude API，只负责推理，不持有任何状态

## 两种运行模式

### 1. 普通对话模式（Interactive Mode）
- 用户通过终端输入 → QueryEngine 处理 → 响应输出
- 权限通过交互式对话框确认

### 2. Coordinator/Worker 模式
- `CLAUDE_CODE_COORDINATOR_MODE` 环境变量激活
- Coordinator：制定计划，通过 `<task-notification>` XML 派发任务
- Worker：接收任务，独立上下文执行，通过摘要回传结果
- INTERNAL_WORKER_TOOLS: {TeamCreate, TeamDelete, SendMessage, SyntheticOutput}

## 关键数据结构（learn-claude-code/data-structures.md）

```
QueryState         → 跨轮次可变状态（turn_count, transition, continuation_count）
TransitionReason   → 继续原因（tool_result_continuation/max_tokens_recovery/compact_retry/transport_retry）
ToolUseContext     → 工具执行环境（handlers/permissions/mcp_clients/messages/app_state/cwd）
ToolResultEnvelope → 标准化结果（ok/content/is_error/attachments）
PermissionDecision → 权限决策结果（behavior: allow/deny/ask）
TaskRecord         → 持久工作图节点（goal/status/dependencies）
RuntimeTaskState   → 运行时执行槽（live executor）
WorktreeRecord     → 隔离执行通道记录
```

## 启动初始化序列

```
1. buildSystemInitMessage()   → 输出会话元数据
2. loadAllPluginsCacheOnly()  → 预加载Skill/插件元数据
3. load_claude_md()           → 加载项目CLAUDE.md
4. load_memory_files()        → 加载Memory Files
5. fetchSystemPromptParts()   → 并行组装系统提示（3路并行）
6. build_system_prompt()      → 最终组装：SystemPrompt + PromptParts
7. → 进入 QueryLoop
```
