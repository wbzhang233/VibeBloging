## 一个请求的完整垂直流

```
用户消息输入
  │
  ▼
processUserInput()          ← 预处理：斜杠命令拦截
  │
  ▼
初始化 QueryState
  │  turn_count = 1
  │  transition = None
  │  continuation_count = 0
  │  has_attempted_compact = False
  ▼
fetchSystemPromptParts()    ← 3路并行：systemPrompt + userContext + systemContext
  │
  ▼
组装 API Payload
  │  system prompt blocks
  │  normalized messages
  │  memory attachments
  │  reminders
  │  tool definitions
  ▼
┌─────────────────────────────────┐
│         QUERY LOOP              │
│                                 │
│  while True:                    │
│    micro_compact(messages)      │  ← 每轮静默压缩旧tool_result
│    if tokens > threshold:       │
│      compress_context()         │  ← 触发压缩策略
│    response = claude_api(msgs)  │  ← 模型推理
│                                 │
│    if response.has_tool_call:   │
│      check_permissions()        │  ← 5阶段权限门
│      run_pre_tool_hooks()       │  ← PreToolUse Hooks
│      result = execute_tool()    │  ← StreamingToolExecutor
│      run_post_tool_hooks()      │  ← PostToolUse Hooks
│      messages.append(result)    │  ← 结果注入
│      state.transition = "tool_result_continuation"
│      continue                   │
│                                 │
│    else:                        │
│      display_output(response)   │
│      run_stop_hooks()           │  ← Stop Hooks
│      user_input = wait_input()  │
│                                 │
└─────────────────────────────────┘
```

---

## 三个核心层（来自 s00b）

```
查询循环层 (Query Loop Layer)
  - 拥有多轮次请求过程
  - 管理 QueryState 和 TransitionReason

工具控制平面 (Tool Control Plane)
  - 拥有工具路由、权限检查、Hooks、执行上下文
  - ToolUseContext 是关键共享环境

平台状态层 (Platform State Layer)
  - 拥有持久记录：任务、运行时槽位、队友、Worktree、外部能力配置
```

## QueryState 可变状态结构

```python
state = {
    "messages": [...],          # 消息历史（内容状态）
    "tool_use_context": {...},  # 工具执行环境
    "turn_count": 1,            # 轮次计数（控制状态）
    "continuation_count": 0,    # 继续计数（控制状态）
    "has_attempted_compact": False,  # 压缩标志（控制状态）
    "max_output_tokens_override": None,  # token上限覆盖
    "transition": None,         # 继续原因（控制状态）
}
```

**关键认识**：内容状态（messages）和控制状态（turn_count, transition等）必须分离。

---

## TransitionReason（继续原因枚举）

```python
TRANSITIONS = [
    "tool_result_continuation",  # 正常：工具结果写回后继续
    "max_tokens_recovery",       # 恢复：输出超token限制后重试
    "compact_retry",             # 恢复：压缩后重试
    "transport_retry",           # 恢复：网络传输失败后重试
]
```

## 工具调用结果的多重影响

一次工具调用不只返回文本，还可能更新：
- 任务板状态（task board state）
- 运行时任务槽状态（runtime task state）
- Memory记录（memory records）
- 请求记录（request records）
- Worktree记录（worktree records）

## doesMostRecentAssistantMessageExceed200k()

当模型输出超过200K上下文时触发 `ESCALATED_MAX_TOKENS` 恢复流程：
- `MAX_OUTPUT_TOKENS_RECOVERY_LIMIT = 3` → 最多尝试3次恢复

## 消息与提示组装管道（s10a 核心）

```
多源输入:
  system prompt blocks  ← 稳定指令块
  messages              ← 对话历史
  memory attachments    ← Memory Files 附件
  reminder injections   ← 临时模式提示（如"当前在Plan Mode"）
  dynamic context       ← 运行时动态内容
  │
  ▼
normalize()  ← 统一结构化
  │
  ▼
final API payload
```

关键认识：**系统提示只是模型输入的一部分，不是全部**。
