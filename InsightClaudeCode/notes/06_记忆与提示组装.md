## 记忆层次体系

```
┌─────────────────────────────────────────────────────────┐
│ Layer 4: Prompt Cache（提示缓存）                       │
│   重复内容 0 费用 · 命中率 40-95% · 节省40-60% tokens  │
├─────────────────────────────────────────────────────────┤
│ Layer 3: Memory Files（持久跨会话记忆）                 │
│   .claude/memory/ · 跨会话 · 本地 · 不提交git          │
│   MEMORY.md 入口 · 200行/25KB 截断 · 嵌套附件          │
├─────────────────────────────────────────────────────────┤
│ Layer 2: Dialog History（动态对话历史）                 │
│   当前会话 · 线性增长 · 70%触发压缩 · 会话结束丢失     │
├─────────────────────────────────────────────────────────┤
│ Layer 1: CLAUDE.md（固定上下文）                        │
│   每次启动自动加载 · 可git提交 · 团队共享               │
│   加载优先级: 当前目录 → 父目录 → ~/.claude/CLAUDE.md  │
└─────────────────────────────────────────────────────────┘
```

## 记忆类型（s09 四类）

| 类型 | 内容 | 示例 |
|------|------|------|
| `user` | 稳定用户偏好 | "偏好 pnpm over npm" |
| `feedback` | 用户重复的纠正/要求 | "不要修改测试快照" |
| `project` | 不从代码能推断的项目事实 | "该目录不能删除，部署依赖它" |
| `reference` | 外部资源指针 | 监控面板URL、规范文档位置 |

**不应存入记忆的内容**：
- 文件树结构（可重新读取）
- 函数签名（代码是truth source）
- 当前任务状态（属于task/plan）
- 临时分支名或PR号（会过时）
- 密钥/凭证（安全风险）

## CLAUDE.md 加载优先级

```
1. 当前工作目录 → ./CLAUDE.md
2. 父目录逐级向上 → ../CLAUDE.md, ../../CLAUDE.md ...
3. 全局 → ~/.claude/CLAUDE.md
4. 项目级 → .claude/CLAUDE.md

多个文件都存在时：全部加载并合并
```

## 信息存储决策树

```
有一条信息需要存储
  │
  ├── 是否需要跨会话持久保存？
  │     │ 否 → 直接在对话中使用（无需存储）
  │     │
  │     └── 是 → 是否是所有人都应遵守的规范？
  │               │ 是 → CLAUDE.md（提交git，团队共享）
  │               │ 否 → Memory Files（.claude/memory/，本地）
  │
  └── 特殊情况：
        可从代码重新推断的信息 → 不需要存储
```

// 3路并行调用
const [systemPrompt, userContext, systemContext] = await Promise.all([
  getSystemPrompt(),    // 核心指令
  getUserContext(),     // 用户级上下文（Memory Files等）
  getSystemContext(),   // 系统级上下文（环境信息等）
])

// 自定义提示完全替换默认提示
if (customPrompt) {
  return customPrompt  // 完全替换，不合并
