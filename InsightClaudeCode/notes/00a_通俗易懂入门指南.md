# Claude Code 通俗易懂入门指南

> **适合人群**：从未使用过 Claude Code 的开发者、业务人员、技术好奇者
> **阅读时间**：15-20 分钟
> **前置知识**：无（会使用电脑即可）

---

## 写在前面：给不同读者的一句话

**如果你是业务人员**——

Claude Code 就像你雇了一位懂技术的全能实习生。你用自然语言告诉他"帮我写一个数据分析脚本"、"把这份报告改成表格"、"搜一下最新的行业数据"，他就会自己动手去做，遇到拿不准的事情还会主动问你。

**如果你是开发者**——

Claude Code 是 Anthropic 出品的 AI 编程 CLI 工具。但它不只是"在终端里问 AI 问题"——它是一套完整的**执行管道**，能让 AI 的推理结果直接影响你的文件系统、Shell 环境和开发工具。读文件、改代码、跑测试、查日志、搜代码库，全都是真实操作，不是模拟。

## 一个故事：你的第一次 Claude Code 体验

假设你刚打开终端，输入了这样一句话：

```
> 帮我在项目根目录创建一个 TODO.md，列出所有还没写测试的函数
```

**接下来发生了什么？**让我们跟着这句话走完全程。

### 第一步：Claude Code 听懂了你的话

你的这句话被发送给 Claude（就是 Anthropic 的 AI 模型）。Claude 不仅理解了你的意思——"找没测试的函数，然后写到文件里"——还自动拆解出了行动计划：

1. 先得知道项目里有哪些代码文件
2. 再找出哪些函数没有对应的测试
3. 最后创建 TODO.md 文件

### 第二步：Claude 开始"动手"

这是 Claude Code 最与众不同的地方：**它不只是告诉你答案，它直接动手做**。

Claude 先用"搜索工具"扫描了你的代码目录，找到 47 个函数。然后用"读文件工具"逐个检查测试文件，交叉对比后发现有 12 个函数缺少测试。

### 第三步：安全门卫介入

当 Claude 准备创建 TODO.md 文件时，系统弹出了一个确认：

```
Claude 想要创建文件 ./TODO.md
[允许] [拒绝] [始终允许此类操作]
```

**这就是权限系统**——Claude 不会偷偷在你的项目里创建、修改或删除任何文件，每一个"写入"操作都需要你的许可。

### 第四步：任务完成

你点了"允许"，Claude 创建了 TODO.md，里面整整齐齐列着 12 个待写测试的函数名、所在文件和行号。

整个过程大约 30 秒。如果手动做，可能要翻十几个文件花 20 分钟。

**这就是 Claude Code 的核心价值：你说需求，它动手干活，你来拍板。**

## 五个核心概念（用生活比喻讲清楚）

Claude Code 的内部结构乍看很复杂，但其实只需要理解五个核心概念，就能搞清楚这台机器是怎么运转的。

### 概念一：Harness（驱动系统）

**生活比喻**：如果 Claude AI 是一台赛车的引擎，那 Harness 就是**整辆赛车**——方向盘、油门、刹车、仪表盘，所有让引擎能上路跑起来的东西。

引擎再强，没有车身和操控系统，它只能在原地轰鸣。

Claude AI 模型再聪明，如果没有 Harness，它只能在对话框里回答问题，不能真正修改你的代码、运行命令、操作文件。

**一句话记住**：Harness 让 AI 从"只会说"变成"能动手"。

```
Claude Code = Claude AI（大脑） + Harness（手脚和神经系统）
```

### 概念二：QueryLoop（永动机循环）

**生活比喻**：想象一个特别能干的人在工作——

```
思考："我应该先看一下项目目录结构"
  ↓
动手：执行搜索，扫描了 200 个文件
  ↓
思考："找到了，接下来需要读 config.ts"
  ↓
动手：打开文件，阅读内容
  ↓
思考："明白了，现在可以写代码了"
  ↓
动手：创建新文件
  ↓
思考："任务完成，可以汇报了"
  ↓
停止，把结果告诉你
```

这个"**思考 → 动手 → 再思考 → 再动手**"的循环，就是 QueryLoop。

它不是只跑一次——它会根据任务需要循环很多次，直到觉得任务完成了才停下来。复杂的任务可能循环几十次，简单的可能一两次就结束。

**一句话记住**：QueryLoop 是 Claude Code 的心跳——"想，做，想，做"不断循环直到任务完成。

### 概念三：工具系统（45 把瑞士军刀）

Claude AI 本身只会"思考和说话"。是 Harness 给它准备了一套工具箱，让它能真正操作你的电脑。

| 工具类别 | 能做什么 | 举个例子 |
|---------|---------|---------|
| 读取工具 | 看文件、看图片、看 PDF | 你让它"看一下 config.ts"，它就真的读了文件内容 |
| 编辑工具 | 改代码、建文件、编辑笔记 | "把第 42 行的变量名改成 camelCase" → 真改了 |
| 搜索工具 | 在代码库里全文搜索、按文件名找 | "找到所有用了 deprecated API 的地方" |
| 执行工具 | 运行 Shell 命令（npm、git、python 等） | "跑一下测试" → 真跑了 npm test |
| 网络工具 | 搜索互联网、抓取网页 | "帮我查一下 React 19 的新 API" |
| 协调工具 | 派出多个 AI "分身"并行工作 | 同时搜五个目录、同时分析三个文件 |

目前 Claude Code 内置了 **45 种以上的工具**，而且可以通过扩展（MCP）接入更多外部工具。

**一句话记住**：工具是 Claude 的"手"——没有工具，AI 只能空谈；有了工具，AI 可以真正动手做事。

### 概念四：权限系统（安全门卫）

你可能会担心：这么能干的 AI，万一它删了我的代码怎么办？

别担心，Claude Code 有一套非常严格的"门卫制度"：

**五种安全模式**：

| 模式 | 像什么 | 谁适合用 |
|------|--------|---------|
| 默认模式 | 正常保安：看文件随便看，改东西得问我 | 日常使用（推荐） |
| 接受编辑 | 老板信任的助理：编辑文件自动通过，危险操作还是问 | 开发高效期 |
| 只读模式 | 博物馆导览员：只能看，不能碰任何东西 | 代码审查 |
| 免打扰 | 全自动机器人：自己判断，不问你 | CI/CD 自动化 |
| 最高权限 | 超级管理员：跳过所有检查 | 仅限极度信任环境 |

不论哪种模式，都遵循一个核心原则：

```
Deny（禁止） > Allow（允许） > 问你

被明确禁止的操作 → 直接拦截，不会执行
被明确允许的操作 → 直接通过，不打扰你
其他操作 → 弹窗问你
```

**一句话记住**：Claude Code 永远不会偷偷做危险的事——要么你事先允许了，要么它会先问你。

### 概念五：记忆系统（三层笔记本）

Claude 不是金鱼——它有一套分层的记忆系统，让它能"记住"和你相关的信息。

**第一层：项目手册（CLAUDE.md）**

每个项目根目录可以放一个 CLAUDE.md 文件，里面写着项目的规则、偏好和约定。就像给新员工看的"入职手册"——每次 Claude 启动都会自动读它。

比如你可以在里面写：

```
- 本项目使用 TypeScript + React
- 代码风格遵循 Airbnb ESLint 规范
- 所有 API 接口在 src/api/ 目录下
- 测试用 Vitest，不要用 Jest
```

这样 Claude 每次干活都知道这些规矩，不需要你重复说。

**第二层：对话草稿纸（当前会话历史）**

Claude 会记住当前对话中你说过的所有话和它做过的所有操作。但这就像草稿纸——**会话结束就丢了**。

如果对话特别长（超过窗口上限的 70%），Claude 会自动"压缩"老的内容，生成一份摘要来节省空间。这就像你把一大堆草稿纸整理成一页总结——细节可能丢失，但核心信息保留。

**第三层：跨会话日记（Memory 文件）**

如果你需要 Claude 永久记住某些事情（比如"我喜欢用 bun 而不是 npm"），它会写入一个专门的 Memory 文件。下次打开新会话时，Claude 会自动读取这个文件。

这就像你的跨会话"日记本"——不管开多少次新对话，Claude 都记得你之前交代的偏好。

**一句话记住**：手册（CLAUDE.md）是团队共享的规矩，草稿纸是当前对话记录，日记是你的私人偏好——三层配合让 Claude 既懂项目又懂你。

## Claude Code 能帮你做什么？

### 面向开发者的典型场景

| 场景 | 你说什么 | Claude 做什么 |
|------|---------|-------------|
| 写新功能 | "给用户表加一个邮箱验证功能" | 读现有代码 → 设计实现方案 → 写代码 → 写测试 |
| Debug | "这个接口为什么返回 500" | 读日志 → 追踪调用链 → 定位 Bug → 给出修复 |
| 重构 | "把这个 Class 组件改成 Hooks" | 分析组件 → 逐步重写 → 保持功能不变 |
| 代码审查 | "看一下这个 PR 的改动" | 拉取 diff → 检查风格/逻辑/安全 → 逐条点评 |
| 写测试 | "给 UserService 补充单元测试" | 分析公开 API → 生成边界测试 → 运行验证 |
| 学习代码库 | "解释一下认证模块是怎么工作的" | 追溯关键文件 → 画出调用链 → 通俗讲解 |
| 运维 | "检查一下 Docker 配置有没有问题" | 读 Dockerfile → 对比最佳实践 → 建议优化 |

### 面向业务人员的典型场景

| 场景 | 你说什么 | Claude 做什么 |
|------|---------|-------------|
| 数据分析 | "分析一下 sales.csv 的月度趋势" | 写 Python 脚本 → 读数据 → 生成图表 → 解读 |
| 文档撰写 | "把这份技术文档改写成客户能看懂的" | 读原文 → 去技术术语 → 重写为通俗版本 |
| 自动化 | "每天自动把 Excel 里的数据同步到数据库" | 设计方案 → 写脚本 → 安排定时任务 |
| 信息搜索 | "最新的 GDPR 合规要求有哪些变化" | 搜索网页 → 提取关键信息 → 整理成报告 |
| 格式转换 | "把这 50 个 Markdown 文件转成 PDF" | 写批量转换脚本 → 执行 → 输出所有 PDF |

### 进阶场景：多 Agent 协作

当任务非常复杂时，Claude Code 可以变身为"项目经理"模式——

它自己当**协调者（Coordinator）**，然后派出多个 **工人（Worker）** 并行工作：

```
你："重构整个认证模块，并更新所有相关测试"

Claude（协调者）：
  ├── Worker 1：分析现有认证模块的结构
  ├── Worker 2：搜索所有依赖认证模块的文件
  ├── Worker 3：检查当前测试覆盖率
  │
  (等三个 Worker 都报告完毕)
  │
  ├── Worker 4：重构核心认证逻辑
  ├── Worker 5：更新所有依赖文件
  └── Worker 6：重写测试用例
```

**一个人指挥，多人同时干活，效率成倍提升。**

## Claude Code 做不到什么？（诚实边界）

好的工具说明不仅告诉你它能做什么，还要坦诚告诉你它做不到什么：

### 1. 不能访问你未授权的资源

Claude Code 运行在你的本地电脑上。它不能访问你无法访问的系统——没有数据库密码，就连不上数据库；没有服务器 SSH 密钥，就登不上服务器。

### 2. 不能替代人做最终决策

Claude 可以帮你分析、建议、实现，但最终"要不要合并这个 PR"、"要不要部署到生产环境"的决定权始终在你手中。

### 3. 对话太长时会遗忘细节

Claude 的"工作记忆"有上限（约 200K Token，相当于约 40-50 万个汉字）。当对话接近这个上限时，它会自动压缩早期内容。这意味着非常早的对话细节可能被概括性地保留，而不是逐字记住。

### 4. 不会联网获取实时数据（默认）

除非你启用了联网工具，否则 Claude 的知识有截止日期。如果你问"今天的股票价格"，它需要调用搜索工具才能给出答案。

### 5. 可能犯错

和所有 AI 一样，Claude 有时会理解错意思、写出有 Bug 的代码、给出不准确的信息。所以——

```
永远不要无条件信任 AI 的输出。
Review 它写的代码，验证它给的信息，测试它做的修改。
```

## 术语速查表

以下是阅读本系列后续深度笔记时会遇到的核心术语，每个用一句话解释：

| 术语 | 一句话解释 |
|------|----------|
| **Harness** | 把 AI 的"思考能力"变成"做事能力"的整套工程系统 |
| **QueryLoop** | "思考→动手→再思考"的核心循环，不断运转直到任务完成 |
| **QueryEngine** | 整个 Claude Code 的中枢调度器，管理一次会话的全部生命周期 |
| **Tool（工具）** | Claude 用来操作真实世界的"手"，比如 Read（读文件）、Edit（改代码）、Bash（执行命令）|
| **stop_reason** | 循环控制信号：`end_turn` = 任务完成可以停了，`tool_use` = 还需要继续动手 |
| **StreamingToolExecutor** | 工具并发执行引擎——安全的工具（如读文件）可以同时执行多个 |
| **Permission Mode** | 权限模式——从"什么都问"到"什么都不问"的五种安全等级 |
| **CLAUDE.md** | 项目级指令文件，告诉 Claude 这个项目的规矩和偏好 |
| **Memory File** | 跨会话记忆文件（MEMORY.md），记住你的个人偏好 |
| **Prompt Cache** | 把重复的指令内容缓存起来，下次调用不用重新算，节省 40-60% 费用 |
| **autoCompact** | 当对话用量达到 70% 上下文窗口时，自动生成摘要来释放空间 |
| **NSIF** | 情感因子公式：方向 × 强度 × 置信度 × 来源权重 × 时间衰减 |
| **Coordinator/Worker** | 多 Agent 模式——一个"项目经理"协调多个"工人"并行干活 |
| **MCP** | Model Context Protocol——让 Claude 连接外部服务器和工具的标准协议 |
| **Hooks** | 事件钩子——在特定时刻（如工具执行前）自动运行你写的脚本 |
| **Skills** | 可复用的任务模板——把复杂工作流打包成一条命令 |
| **Token** | AI 处理文本的最小单位，约等于 0.75 个英文单词或 0.5 个汉字 |
| **Context Window** | AI 一次能"看到"的全部内容上限，Claude 约 200K Token |
| **Platt Scaling** | 对 AI 输出的"置信度"进行校准的统计方法（AI 往往过度自信） |
| **IC (Information Coefficient)** | 衡量预测因子与真实结果相关性的指标，越高越好 |

## 最后

Claude Code 不神秘。

它的本质就是：**一个极其聪明的 AI 大脑 + 一套精心设计的执行系统 + 严格的安全围栏**。

大脑负责思考，执行系统负责动手，安全围栏保证不失控。

三者配合，让一句自然语言的指令变成对真实世界的可靠操作。

现在你已经了解了这台机器的全貌——接下来不妨打开一张可视化图，或者翻开一篇深度笔记，看看引擎盖下面究竟是什么样的精密结构。
