Claude Code = 一位懂技术的全能实习生
你只需要说需求，它来动手做

它能帮你做这些事情：

  • "帮我分析这份 Excel 数据的趋势" → 自动写脚本、跑分析、出图表
  • "把这份技术文档改写成客户能看懂的" → 读懂原文、去术语、重新组织
  • "搜一下最新的行业政策变化" → 上网搜索、提取关键信息、整理成报告
  • "每天自动把数据同步到数据库" → 设计方案、写自动化脚本、安排定时任务
Claude Code = AI 推理 + 真实世界执行管道
不是聊天框，是完整的 Harness 工程体系

核心技术价值：

  • 45+ 内置工具：Read / Write / Edit / Bash / Grep / Glob / WebSearch ...
  • QueryLoop 循环：API 调用 → 工具执行 → 结果返回 → 再调用，直到 end_turn
  • 5 种权限模式：从 plan（只读）到 bypassPermissions（最高权限）
  • Coordinator/Worker 多 Agent 架构：复杂任务自动拆分并行
  • MCP 协议扩展：接入任意外部服务器和自定义工具
  • Prompt Cache 优化：相同项目目录下缓存命中率 60-95%，成本降低 40-60%

⌨️
第一步：你输入一句话
在终端里用自然语言描述你的需求
（点击展开详情）
例如："帮我找出项目中所有没写测试的函数，列成 TODO 清单。"

Claude Code 接收你的文字后，会把它包装成一条消息发给 Claude AI 模型。这条消息不仅包含你说的话，还自动附带了项目信息（来自 CLAUDE.md）、环境信息（操作系统、当前目录）和你的历史偏好。
🧠
第二步：AI 理解并拆解任务
Claude 分析你的意图，制定行动计划
（点击展开详情）
Claude 不会直接给你一个"答案"——它会思考："要完成这个任务，我需要做什么？"

比如分析出需要：1) 先搜索所有代码文件 → 2) 找出函数定义 → 3) 对比测试文件 → 4) 创建 TODO.md。

这个"思考"过程在 Claude 内部完成，你会看到它开始一步步采取行动。
🔧
第三步：选择并使用工具
从 45+ 个工具中选择合适的，真正动手操作
（点击展开详情）
Claude 会调用具体的工具来执行操作：
• Glob — 按文件名模式搜索（如 "找所有 .ts 文件"）
• Grep — 在代码中全文搜索关键词
• Read — 打开并阅读文件内容
• Write — 创建新文件
• Bash — 运行命令行命令

只读工具（如 Read、Grep）可以并行执行，提高效率；
写入工具（如 Write、Edit）必须串行执行，确保安全。
🛡️
第四步：安全检查
每个操作都要过"门卫"审核——特别是写入和执行类操作
（点击展开详情）
在执行任何可能修改你文件或系统的操作前，权限系统会进行检查：

1️⃣ 首先看有没有禁止规则（比如"不许删除 node_modules"）→ 有就直接拦截
2️⃣ 再看有没有允许规则（比如"允许编辑 src/ 目录"）→ 有就自动通过
3️⃣ 都没有 → 弹窗问你："Claude 想创建文件 TODO.md，允许吗？"

你可以选择：允许这一次 / 始终允许此类操作 / 拒绝。
✅
第五步：执行完毕，返回结果
循环结束，Claude 将结果呈现给你
（点击展开详情）
当 Claude 判断任务已完成时，QueryLoop 退出循环，将最终结果展示给你。

如果任务还没做完（比如搜索完了还没写文件），它会自动回到第二步继续思考下一步操作。

这就是 QueryLoop 的精髓：思考 → 动手 → 思考 → 动手，循环往复直到任务完成或预算用完。

🔒 安全感：三道门
每个操作都要过这三道安全检查，顺序判断，不可跳过
操作请求 → 禁止规则? → 允许规则? → 问你 → 执行/拒绝
🚫
第一道：禁止规则
命中 Deny 规则 → 直接拦截
无论如何不会执行
例如：禁止删除 .env 文件
✅
第二道：允许规则
命中 Allow 规则 → 自动通过
不打扰你
例如：允许读取 src/ 目录
🙋
第三道：问你
两条规则都没命中
→ 弹窗让你做决定
"允许 / 始终允许 / 拒绝"
核心承诺：Claude Code 永远不会偷偷做危险操作——要么你事先允许了，要么它会先问你。
