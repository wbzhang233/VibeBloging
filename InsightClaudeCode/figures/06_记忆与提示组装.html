LAYER 3 · 持久跨会话
Memory Files
src/memdir/memdir.ts
跨会话保存用户偏好、项目约束、调试经验。MEMORY.md 作为入口，支持嵌套引用其他记忆文件。
ENTRYPOINT = 'MEMORY.md' MAX_LINES = 200 MAX_BYTES = 25KB 本地不提交git TEAMMEM gate EXTRACT_MEMORIES gate
路径: .claude/memory/
入口: MEMORY.md（截断200行/25KB）

记忆类型:
user → 用户偏好（prefer pnpm）
feedback → 重复纠正（勿改快照）
project → 项目约束（旧目录不可删）
reference → 外部链接（监控面板）

团队记忆: TEAMMEM gate
自动提取: EXTRACT_MEMORIES gate
LAYER 2 · 动态上下文
Dialog History
QueryEngine.mutableMessages
当前会话的对话历史和工具调用结果，随交互线性增长。超过70%上下文窗口时触发压缩策略。会话结束后丢失（需通过SessionMemory持久化）。
线性增长 70%触发压缩 会话结束丢失 FileStateCache跟踪
mutableMessages: Message[] ← 核心状态
loadedNestedMemoryPaths: Set<string>
discoveredSkillNames: Set<string>

SessionMemory: src/services/SessionMemory/
会话结束时将关键决策持久化
存入: .claude/session-memory/{ts}.md
LAYER 1 · 固定上下文
CLAUDE.md
每次启动自动加载
项目规范和约定，每次启动自动加载，可提交到 git 实现团队共享。加载时按目录层级向上查找，多个文件全部加载合并。
每次加载 可git提交 团队共享 /memory命令编辑

../CLAUDE.md 父目录

信息存储决策树
有信息需要存储?
  │
  ├── 跨会话持久? → 否 → 直接在对话中使用
  │
  └── 是 → 所有人遵守的规范?
              │ 是 → CLAUDE.md (提交git)
              │ 否 → Memory Files (本地)

⚠ 可从代码推断的信息 → 不需要存储
