Sub-Agent（Agent 工具派生）
一次性任务委派。完成返回摘要，然后销毁。无独立身份，无跨会话记忆。
独立上下文 Explore/Plan/Execute 角色 摘要返回
模式 B：Coordinator / Worker 双模架构
CLAUDE_CODE_COORDINATOR_MODE=1 激活 · Feature Gate
Coordinator（协调者）
Coordinator Agent
600行+专用系统提示。制定计划，分解任务，通过 task-notification XML 向 Workers 派发任务，汇聚结果。
getCoordinatorSystemPrompt() getCoordinatorUserContext() isCoordinatorMode() scratchpadDir 注入
<task-notification> XML
↓↓↓
Worker A · Explore
探索代理
✓ Read ✓ Glob/Grep ✓ WebFetch ✗ Write/Bash
Worker B · Plan
规划代理
✓ Read/Glob ✓ TodoWrite ✓ TaskCreate ✗ Write/Bash
Worker C · Execute
执行代理
✓ 全部工具 ✓ Bash/Write ✓ Agent ✓ SyntheticOutput
↑
结果汇聚
SyntheticOutput → Coordinator
Workers 通过 SyntheticOutput 工具将结果传回 Coordinator。Coordinator 汇聚所有摘要，生成最终响应。
task-notification XML 协议（Coordinator → Worker 通信）
Coordinator 向 Worker 发送的任务通知（通过 user-role 消息携带）
<task-notification>
  <task-id>task_001</task-id>
  <instructions>
    分析 src/auth/ 目录下所有文件
    生成安全审查报告
  </instructions>
  <allowed-tools>
    Read, Grep, Glob, WebFetch
  </allowed-tools>
  <output-format>markdown</output-format>
  <scratchpad-path>.claude/scratch/</scratchpad-path>
</task-notification>
INTERNAL_WORKER_TOOLS（Worker 专用工具集）

三种 Agent 类型对比
s04 · One-Shot
Sub-Agent
一次性派发，完成返回摘要
无独立身份，无跨会话记忆
Agent 工具（原名 Task）派生
独立上下文窗口
可选 git worktree 隔离
角色: Explore / Plan / Execute
s15 · Persistent
Teammate（持久队友）
长期存活，有独立身份和角色
JSONL 收件箱（append-only）
每次LLM调用前检查收件箱
状态: spawn→WORKING→IDLE→...
存储: .team/config.json
通信: MessageBus.send() / read_inbox()
s17 · Autonomous
Autonomous Agent（自主代理）
自主认领任务，无需人工派发
基于任务板协调机制
WorktreeRecord 独立执行通道
TaskRecord 驱动（durable goal）
RuntimeTaskState 实时执行槽
ScheduleCronTool 支持定时触发
Worktree 生命周期（git worktree 隔离执行）
创建
git worktree add .claude/worktrees/{name}
→
派发
Worker在独立目录执行
→
执行
文件改动不影响主目录
→
汇聚
SyntheticOutput摘要返回Coordinator
→
清理
git worktree remove或保留分支
TaskRecord (durable goal) ≠ RuntimeTaskState (live executor) · 两者分离是系统稳健性的关键设计
