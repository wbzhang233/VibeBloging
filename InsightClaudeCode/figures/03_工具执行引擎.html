T+0ms T+50ms T+100ms T+200ms T+500ms T+1000ms
场景A: 并发安全工具（isConcurrencySafe = true）→ 并行执行
Read(main.ts) safe
Read · 250ms
Read(utils.ts) safe
Read · 180ms
Grep("class") safe
Grep · 300ms
↑ 三个工具同时开始执行，总耗时 = max(250, 180, 300) = 300ms
场景B: 非并发工具（isConcurrencySafe = false）→ 串行独占执行
Bash(npm build) blocking
Bash · 500ms（独占执行）
Write(output.js) waiting
Write（等待Bash完成）
↑ Bash 独占执行期间，Write 必须等待。siblingAbortController: Bash失败 → 终止所有兄弟子进程
结果排序：并发执行，但结果按原始请求顺序缓冲排序后输出
请求顺序: [Read(a), Read(b), Grep(.)] → 完成顺序: [Read(b)✓, Grep(.)✓, Read(a)✓] → 输出顺序: [Read(a), Read(b), Grep(.)] ← 按请求序

// Layer 1: ToolSpec（模型看到的）
tool = {
  name: "read_file",
  description: "Read file contents",
  input_schema: {...}
}

// Layer 2: Tool Router（路由决策）
if name.startsWith("mcp__"): → MCP Client
elif name == "Agent":       → SubAgent Runner
elif name == "task_create": → Task System
else:                        → Native Handler

// Layer 3: ToolUseContext（关键！共享环境）
ctx = {
  tools: handlers_map,
  permission_context: {...},
  mcp_clients: {...},
  messages: current_msgs,   // 可读
  app_state: {...},
  notifications: [],
  cwd: "/path/to/project"
}

// Layer 4: ToolResultEnvelope（标准化返回）
result = {
  ok: True,
  content: "...",
  is_error: False,
  attachments: []
}
单次工具调用完整管道
// 1. 模型输出工具调用
{type:"tool_use", name:"Bash", input:{"command":"ls"}}

// 2. 工具路由
router.route("Bash") → native_handler

// 3. 权限检查（5阶段）
permission_result = useCanUseTool(ctx)
if denied: return error

// 4. PreToolUse Hooks
pre_result = await runPreToolHooks(ctx)
if pre_result.blocked: return blocked_error

// 5. 实际执行（StreamingToolExecutor）
tracked = TrackedTool(isConcurrencySafe=False)
result = await executor.execute(tracked, ctx)

// 6. PostToolUse Hooks
await runPostToolHooks(result, ctx)

// 7. 结果注入
messages.append({
  role:"tool", id:tool_use_id, content:result
})
