6种压缩策略（按触发阶段排序）
0
工具执行时触发
persistedOutput
工具输出超阈值时写入磁盘，消息中替换为预览标记。第一道防线，在输出进入上下文之前拦截。
触发阈值: 通用 50KB / Bash 30KB
写入路径: .task_outputs/tool-results/{id}.txt
消息替换: <persisted-output>
Output too large (48.8KB). Saved to: ...
Preview (first 2000 chars): ...
</persisted-output>
后续: 模型可用 Read 读取完整内容
1
每轮推理前静默触发
microCompact
每次 LLM 调用前自动扫描，将3轮以前的旧 tool_result 替换为占位符。对用户完全不可见。
扫描: 所有 tool_result 条目
保留: read_file 结果（参考材料）
替换: "[Previous: used {tool_name}]"
策略: 仅替换3轮以前的结果
效果: 无感知地清理过期工具输出
2
tokens > AUTOCOMPACT_THRESHOLD
autoCompact
Feature Gate: 默认开启
主要自动压缩策略。先保存完整 transcript，再用 LLM 摘要整个对话，替换所有消息。
常量:
AUTOCOMPACT_BUFFER_TOKENS = 13,000
WARNING_THRESHOLD_BUFFER = 20,000
MAX_OUTPUT_TOKENS_FOR_SUMMARY = 20,000
MAX_CONSECUTIVE_FAILURES = 3 (断路器)

流程:
1. 保存 transcript → .transcripts/{ts}.jsonl
2. LLM摘要整个对话（80K字符上限）
3. messages = [{summary}]

环境变量覆盖:
CLAUDE_CODE_AUTO_COMPACT_WINDOW
CLAUDE_AUTOCOMPACT_PCT_OVERRIDE
3
模型主动调用 compact 工具
manualCompact
模型自主判断时机，主动调用 compact 工具触发。与 autoCompact 完全相同的机制，区别在于决策者是模型而非阈值。
触发者: 模型（而非阈值触发）
机制: 与 autoCompact 完全相同

适用场景:
- 完成阶段性任务后主动清理
- 预判即将进行大量工具调用前
- 模型认为当前历史冗余较多时
4
HISTORY_SNIP gate 激活
snipCompact
Feature Gate: HISTORY_SNIP
外科手术式压缩。不是摘要整个对话，而是精准裁剪历史中占用空间最大的工具结果片段。
策略: 外科手术式裁剪（非全量摘要）
定位: 历史中最大的工具结果
操作: 精准替换而非整体摘要
优势: 保留更多历史结构与脉络
vs autoCompact: 更保守，侵入性更小
5
REACTIVE_COMPACT gate 激活
reactiveCompact
Feature Gate: REACTIVE_COMPACT
响应式压缩策略。在检测到上下文压力时主动响应，不等到阈值达到才触发。
触发: 检测到上下文压力信号
策略: 响应式，比 autoCompact 更主动
目标: 在问题严重前提前干预
配合: QueryState.has_attempted_compact
防止同轮次重复压缩
6
CONTEXT_COLLAPSE gate 激活
contextCollapse
Feature Gate: CONTEXT_COLLAPSE
极端情况下的完全上下文折叠。当其他策略都无法解决问题时使用的最后手段。
触发: 极端情况（多次压缩失败后）
策略: 完全折叠，保留最少必要信息
场景: 超长会话 / 循环工具调用
状态: has_attempted_compact = True
transition = "compact_retry"
压缩触发状态机
正常对话
│ tokens 持续增长
▼
[WARNING_THRESHOLD_BUFFER_TOKENS = 20K 前]
│ → 显示警告提示给用户
▼
[AUTOCOMPACT_THRESHOLD] ← getAutoCompactThreshold(model) 计算
│ 触发 autoCompact
▼
[成功] → 继续, has_attempted_compact = True
│
▼ 3次连续失败 (断路器)
│ MAX_CONSECUTIVE_AUTOCOMPACT_FAILURES = 3
▼
[放弃压缩] → 报错或以原始内容继续
QueryState 压缩相关字段 + 恢复流程
// QueryState 压缩控制字段
has_attempted_compact: False ← 防同轮重复压缩
transition: "compact_retry" ← 压缩后继续原因
max_output_tokens_override: N ← 压缩期间缩减输出

// TransitionReason 四种枚举
"tool_result_continuation" ← 正常工具结果继续
"max_tokens_recovery" ← 超200K恢复
"compact_retry" ← 压缩后重试
"transport_retry" ← 网络失败重试

// 恢复常量
ESCALATED_MAX_TOKENS
MAX_OUTPUT_TOKENS_RECOVERY_LIMIT = 3
