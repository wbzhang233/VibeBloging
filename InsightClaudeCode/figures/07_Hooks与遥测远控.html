Hooks 事件总线（7种事件类型 · 2种内部专用）
可配置 · 可阻止
PreToolUse
工具执行前触发
最重要的 Hook。每次工具调用前触发，是整条链中唯一能阻止工具执行的 Hook。支持 matcher 过滤特定工具。可用于安全审计、危险命令拦截、前置验证。
exit 0 → 继续执行工具
exit 1 → 阻止，stderr→错误
exit 2 → 注入对话，仍执行
JSON {decision:"block"} → 高级阻止
可配置 · 不可阻止
PostToolUse
工具执行后触发
工具执行完成后触发。不能阻止已完成的操作，但可以追加内容到工具结果。用于日志记录、自动格式化、运行测试（写文件后自动运行 lint）。
exit 0 → 正常继续
exit 2 → stderr追加到结果
HookAttachment:
hook_cancelled
hook_blocking_error
hook_output
可配置
Stop
模型最终输出时
模型停止生成并输出最终结果时触发。用于发送通知（"Claude已完成"）、自动保存会话摘要、触发后续自动化流程。
SessionStorage.flushSessionStorage()
通知推送
自动保存
可配置
TaskCompleted
任务完成时
Task 系统中的任务完成时触发（区别于 Stop）。用于任务完成回调、触发下游工作流。
TaskRecord.status → completed
结果回调
可配置
TeammateIdle + Notification
队友空闲时 / 需要关注时
TeammateIdle: 持久队友进入空闲状态，可用于资源回收或分配新任务。NotifyAfterTimeout: 长时间运行任务超时后的通知 Hook，避免用户等待不知情。
UserPromptSubmit:
用户消息进入时
输入预处理
⚠ 内部专用 · 不可外部配置
PostSampling
模型采样完成后（内部）
内部私有 Hook，不暴露在 settings.json 配置中。驱动 AutoDream、PromptSuggestion、EXTRACT_MEMORIES 等内部高级功能。仅 Anthropic 内部逻辑使用。
AutoDream → 提示建议生成
PromptSuggestion → 对话建议
EXTRACT_MEMORIES → 自动记忆提取
JobClassifier → 任务分类

Hooks 配置示例
// settings.json 中的 hooks 配置
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash",
        "command": "python /path/to/audit.py"
      },
      {
        "matcher": "Write",
        "command": "/path/to/lint-check.sh"
      }
    ],
    "Stop": [
      {
        "command": "notify-send 'Claude Done'"
      }
    ]
  }
}
Hook 执行环境变量
# Hook 脚本接收的环境变量
HOOK_EVENT=PreToolUse
HOOK_TOOL_NAME=Bash
HOOK_TOOL_INPUT='{"command":"ls -la"}'
HOOK_TOOL_OUTPUT='...'  # PostToolUse only

# Matcher 规则
"matcher": "Bash"        # 精确匹配
"matcher": "Bash(git *)" # 参数模式
# 无matcher → 匹配所有工具
