### 部署架构对比

```
本地部署（QClaw / WorkBuddy / CoPaw / AutoClaw）
优点：数据不出本地，隐私性强，可完全离线
缺点：电脑关机任务停，安全防护靠自己

云端托管（ArkClaw / 飞书OpenClaw / Kimi Claw）
优点：7×24 在线，厂商负责安全，不消耗本地资源
缺点：数据在云上，深度绑定特定生态

端云双部署（CoPaw / JVS Claw）
优点：灵活选择，本地可操控真实桌面应用
缺点：配置复杂，稳定性取决于用户自身
```

## 1. 背景：什么是"AI龙虾"热潮

2026 年 3 月，国内 AI 应用市场掀起一场"全民养虾"热潮。腾讯、字节、阿里、百度、小米、智谱、月之暗面 7 大巨头，在短短两周内密集发布共计 **12 款** AI 智能体产品。

产品图标均形似龙虾，统称"Claw 系列"，均基于 **OpenClaw** 开源框架构建。

### 本质是什么？

```
AI 发展路径：

第一代：AI 聊天助手（问答对话）
    → 你问，AI 答

第二代：AI 代码补全（IntelliSense Plus）
    → AI 帮你写下一行代码

第三代：AI 智能体 / Claw 系列（当前阶段）
    → AI 作为"数字员工"，自主规划并执行复杂任务
```

"养虾"的本质，是 **AI Agent 从极客玩具走向大众应用的关键节点**。

## 2. OpenClaw 框架解析

所有 Claw 产品的底座，是 **OpenClaw 开源框架**（类比：Claude Code 是 Anthropic 官方实现，OpenClaw 是社区生态的中国版）。

### 核心能力

| 能力维度 | 描述 |
|---------|------|
| 任务规划 | 将自然语言需求拆解为可执行步骤序列 |
| 工具调用 | 操控浏览器、文件、应用程序、API |
| 多 Agent 协作 | 多个 AI 并行分工，协同完成复杂任务 |
| 技能扩展 | 通过 Skills/插件扩展能力边界 |
| 跨平台执行 | 支持本地部署 / 云端托管两种模式 |

### 重大安全事件（背景知识）

> ⚠️ 2026 年 3 月，安全机构披露 OpenClaw 高危漏洞
> **CVE-2026-25253，CVSS 评分 8.8**
> 影响范围：**超过 93.4% 的公网实例可被绕过认证**
>
> 这直接影响了各厂商的产品定位选择：本地部署 vs 云端隔离。

## 3. 腾讯系产品

腾讯一次性推出 **3 款**产品，形成完整的"用 + 管"矩阵。

### 3.1 QClaw — 零门槛入门款

**官网**：https://qclaw.qq.com/
**发布**：2026 年 3 月 9 日（3 月 24 日全面开放，无需邀请码）
**开发团队**：腾讯电脑管家团队

#### 产品定位

把 OpenClaw 从"需要配置环境的极客工具"变成"双击安装的傻瓜产品"，主打**小白零门槛上手**。

#### 核心特色

- **一键安装**：无需命令行，无需配置环境，双击安装包即完成
- **微信遥控**：绑定微信后，在微信发消息即可远程指挥电脑干活
- **可视化龙虾工作室**：任务执行过程拟人化呈现——AI "虾仔"会思考、敲键盘、报错时掉眼泪，增强趣味性和可观察性
- **全面开放**：无需邀请码，直接注册使用

#### 适合人群

普通职场人、微信/企微重度用户、不想折腾配置的小白

#### 评价

> "OpenClaw 的平民化入口，把技术门槛拉到地板。"

---

### 3.2 WorkBuddy — 职场办公款

**官网**：https://www.codebuddy.cn/work/
**发布**：2026 年 3 月 9 日
**开发团队**：腾讯云 CodeBuddy 团队

#### 产品定位

面向职场场景的 **AI 原生桌面智能体工作台**，主打"一句话下任务，AI 自主交付结果"。

#### 核心特色

- **全场景覆盖**：文档编辑、表格制作、PPT 生成、数据分析，一站式搞定
- **多平台联动**：支持通过企业微信、QQ、飞书、钉钉远程操控
- **跨系统支持**：Mac / Windows 双平台
- **通勤场景优化**：路上发语音，到公司直接拿成果

#### 实测表现（引用第三方评测）

| 测试场景 | 表现 |
|---------|------|
| 简单任务（生成资讯网页） | ✅ 表现尚可 |
| 复杂任务（完整市场分析报告） | ⚠️ 中途崩溃 2 次 |
| 首发当天稳定性 | ❌ 服务崩溃，扩容 10 倍后恢复 |

#### 适合人群

职场人士、中小企业、需要融入企业微信/QQ 办公生态的团队

#### 评价

> "能用，但再等等它长大一点会更好。"

### 3.3 龙虾管家 — 安全防护款

**官网**：https://guanjia.qq.com/main.html

#### 产品定位

**不是一只"虾"，而是给所有"虾"提供安全防护的"虾笼"**，解决 AI Agent 使用中的安全隐患。

#### 核心功能

| 功能 | 说明 |
|------|------|
| 文件夹禁区 | 设定敏感目录，AI 无法访问/修改 |
| 插件安全检测 | 自动检测恶意插件，防止供应链攻击 |
| 防公网暴露 | 避免端口配置不当导致的外部入侵 |
| 操作日志回溯 | 所有 AI 操作留有完整审计日志 |
| 沙箱隔离 | 将 AI 运行在隔离环境，防止越权操作 |

#### 适合人群

**所有 OpenClaw 用户**，尤其是企业/团队环境，强烈建议必装。

## 4. 字节系产品

字节押注**云端托管**路线，推出 2 款产品，主打省心稳定。

### 4.1 ArkClaw — 云端托管首选

**官网**：https://www.volcengine.com/product/arkclaw
**依托平台**：火山引擎

#### 产品定位

**不需要本地部署的云端"养虾"方案**——订阅火山引擎套餐，免费获得一台预装 OpenClaw 的云主机。

#### 定价

| 套餐 | 价格 | 适合人群 |
|------|------|---------|
| Lite 套餐 | 首月 9.9 元（体验 7 天） | 想先试试水的用户 |
| Pro 套餐 | 首月 49.9 元（订阅期间持续可用） | 长期使用用户 |

#### 核心优势

- **Token 免焦虑**：实测高强度使用 3 天，Token 消耗仅占一周总量的 23%
- **稳定性极强**：3 天高强度测试，零卡顿零崩溃
- **安全隔离**：AI 运行在云端虚拟机，从源头隔离本地数据泄露风险，直接规避 CVE-2026-25253 漏洞影响
- **多 Agent 协作**：实测 4 个 AI 并行分工——信息搜集、分析、生成报告、自动同步飞书，全程流畅
- **7×24 在线**：电脑关机后云端任务继续运行

#### 局限性

- 深度绑定飞书生态，非飞书用户体验打折
- 数据托管在云端，对数据主权敏感的企业需评估

#### 评价

> "三款（WorkBuddy/ArkClaw/CoPaw）里产品完成度最高，买它等于买省心。"

### 5.1 JVS Claw（贾维斯）— 手机端首选

**开发团队**：阿里云无影团队

- **手机原生**：三步操作即可拥有即开即用的 AI 智能体

- **可视化操控**：AI 执行过程实时可见，遇到卡点可手动介入
- **邀请制内测**：App 内直接申请邀请码

#### 适合人群

手机重度用户、经常移动办公的职场人

### 5.3 CoPaw / QoderWork — 开发者工具

**CoPaw 官网**：https://copaw.agentscope.io/
**QoderWork 官网**：https://qoder.com/qoderwork

#### 产品定位

面向**开发者**的轻量级开源 AI 智能体工具，基于阿里 AgentScope 框架，主打极致轻量和本地操作能力。

#### CoPaw 核心特色

| 维度 | 数据 |
|------|------|
| 内存占用 | ~150MB（标准 OpenClaw Worker 约 500MB 的 **1/5**） |
| 并发优势 | 同等资源可运行 5 倍数量的 Agent |

- 消息回复无故中断

- 实测出现 CPU 占用 166% 的残留进程

#### 评价

> "潜力巨大，但目前还是'开发者玩具'，普通用户慎入。"

## 8. 安全性专题

安全是本轮 Claw 产品竞争的**核心战场**之一。

### 8.1 三种安全架构对比

```
本地部署（QClaw / CoPaw / AutoClaw）
┌─────────────────┐
│   用户电脑       │  ← 数据完全在本地，隐私性最强
│   AI 运行环境    │     但安全防护责任在用户自己
└─────────────────┘

云端托管（ArkClaw / Kimi Claw）
┌─────────────────────────────┐
│   云端虚拟机（隔离环境）      │  ← 数据在云上，厂商负责安全
│   AI 运行环境                │     天然规避本地漏洞利用
└─────────────────────────────┘

端云双部署（CoPaw）
┌─────────┐   ┌─────────────┐
│  本地    │ + │  云端        │  ← 用户自由选择，灵活性最高
└─────────┘   └─────────────┘
```

### 8.2 CVE-2026-25253 漏洞影响评估

| 产品 | 架构 | 漏洞影响 | 应对方式 |
|------|------|---------|---------|
| QClaw | 本地部署 | 有风险 | 依赖龙虾管家防护 |
| WorkBuddy | 本地部署 | 有风险 | 龙虾管家沙箱隔离 |
| **ArkClaw** | **云端隔离** | **基本规避** | **云端虚拟机从源头隔离** |
| CoPaw | 本地/云端 | 本地模式有风险 | 用户自行承担 |
| AutoClaw | 本地部署 | 有风险 | 依赖用户自身防护 |

### 8.3 企业用户安全建议

1. **必装龙虾管家**（腾讯系产品）
2. **云端托管优先**（ArkClaw）规避本地漏洞攻击面
3. **设置文件禁区**：将财务数据、合同文档等敏感目录列入黑名单
4. **不要将 OpenClaw 端口暴露到公网**
5. **保留操作审计日志**，便于事后溯源

## 10. 总结与展望

### 三大厂商的路线选择

| 厂商 | 核心赌注 | 逻辑 |
|------|---------|------|
| **腾讯** | 社交入口 | 11 亿微信用户是最大的分发渠道 |
| **字节** | 云端体验 | 稳定性+安全性+飞书生态，To B 优先 |
| **阿里** | 开源生态 | 开发者优先，建立技术话语权 |

### 当前阶段的共同问题

1. **产品成熟度参差不齐**：首发即崩溃（WorkBuddy）、稳定性差（CoPaw）是普遍现象
2. **生态绑定过重**：飞书绑定、钉钉绑定、微信绑定，用户迁移成本高
3. **安全风险未完全解决**：CVE-2026-25253 对本地部署产品影响持续
4. **复杂任务仍不稳定**：当前阶段适合简单到中等复杂度的任务

### 判断维度总结

```
这波"AI龙虾"热潮的本质，不是模型更强了，
而是 AI 从"帮你想"进化到了"帮你做"。

选型的核心问题只有三个：
1. 你在哪个生态里工作？（微信/飞书/钉钉）
2. 你有没有技术能力折腾？
3. 你更在意数据安全还是使用便捷？

回答这三个问题，答案自然就有了。
```
