## 1. 什么是多智能体框架

单 Agent 模式：用户 -> [一个 LLM + 工具集] -> 结果

多 Agent 模式：
  用户 -> [研究员 Agent] -> [分析师 Agent] -> [审核员 Agent] -> 结果
               搜索工具              计算工具               写作工具

**为什么需要多 Agent？**

| 问题 | 单 Agent | 多 Agent |
|------|---------|----------|
| 上下文窗口限制 | 一个大任务容易超限 | 每个 Agent 专注子任务 |
| 任务质量 | 万金油，每步都平庸 | 专家分工，各步骤精准 |
| 自我校验 | 难以发现自身错误 | 不同 Agent 互相 review |
| 并行处理 | 顺序执行 | 多 Agent 并发加速 |

多智能体框架要解决的核心问题：**如何让多个 AI Agent 高效协作，完成单个 Agent 无法胜任的复杂任务。**

## 2. 框架全景图

| 维度 | 说明 |
|------|------|
| **协作模式** | 顺序执行 / 并行 / 自由对话 / 层级监督 |
| **Agent 粒度** | 通用 Agent vs 角色专家 vs 流程节点 |

各框架定位一览：

- **MetaGPT**：软件工程专用，SOP 最完整
- **LangGraph**：最灵活，状态图控制最精细

## 3. AgentScope：阿里开源的工业级多 Agent 框架

### 3.1 框架简介

- **GitHub：** modelscope/agentscope（⭐ 7k+）
- **语言：** Python
- **发布：** 2024 年 1 月（阿里巴巴达摩院）
- **定位：** 面向工业级多 Agent 协作的研究与生产框架
- **论文：** AgentScope: A Flexible yet Robust Multi-Agent Platform (arXiv 2024)

AgentScope 的核心设计哲学：**消息驱动（Message-Driven）+ Actor 模型**，让多个 Agent 之间的通信像发消息一样自然。

### 3.2 核心架构（五层）

| 层级 | 内容 |
|------|------|
| 应用层 | 角色对话 / 流水线 / 分布式 Agent 系统 |
| Agent 层 | DialogAgent / ReActAgent / UserAgent / 自定义 |
| 消息层 | 统一格式：name + content + metadata |
| 模型层 | OpenAI / Claude / Qwen / Ollama / 本地模型 |
| 服务层 | 工具调用 / RAG / 代码执行 / 搜索 / 自定义 |

### 3.3 基础用法：多 Agent 顺序协作

```python
import agentscope
from agentscope.agents import DialogAgent
from agentscope.msghub import msghub

agentscope.init(model_configs=[{
    "model_type": "openai_chat",
    "config_name": "gpt4o",
    "model_name": "gpt-4o",
}])

researcher = DialogAgent(
    name="研究员",
    model_config_name="gpt4o",
    sys_prompt="你是市场研究专家，负责收集和梳理信息，输出结构化研究报告。"
)

analyst = DialogAgent(
    name="量化分析师",
    model_config_name="gpt4o",
    sys_prompt="你是量化分析师，基于研究报告提炼量化信号和交易逻辑。"
)

critic = DialogAgent(
    name="风控审核员",
    model_config_name="gpt4o",
    sys_prompt="你是风控专家，负责识别分析中的漏洞和风险点，提出改进建议。"
)

with msghub([researcher, analyst, critic]) as hub:
    msg = researcher({"content": "调研当前AI Agent市场竞争格局"})
    msg = analyst(msg)
    final = critic(msg)
```

### 3.4 原生分布式 Agent：Scale-Out 能力

AgentScope 最独特的能力是**框架级原生分布式支持**：

```python
researcher = DialogAgent(
    name="研究员",
    to_dist=True,        # 只需一个参数，Agent 变成分布式服务
    host="192.168.1.10",
    port=12010
)

# 多机并发
from agentscope.pipelines import parallel_pipeline

results = parallel_pipeline(
    agents=[researcher_cn, researcher_en, researcher_jp],
    x={"content": "调研AI量化投研工具市场"}
)
```

### 3.5 Pipeline 组合模式

```python
from agentscope.pipelines import (
    sequential_pipeline,   # 顺序
    parallel_pipeline,     # 并行
    whileloop_pipeline,    # 循环直到满足条件
    ifelse_pipeline,       # 条件分支
    forloop_pipeline,      # 遍历执行
)

# 并行调研 -> 汇总 -> 循环审核（最多 3 次）
def not_approved(msg):
    return "通过" not in msg.content

pipeline = sequential_pipeline([
    lambda x: parallel_pipeline([cn_researcher, en_researcher], x),
    summarizer,
    lambda x: whileloop_pipeline(critic, x, not_approved, max_loop=3)
])
```

### 3.6 优缺点分析

**优点：**
- 原生分布式支持（Actor 模型），框架级能力而非插件
- Pipeline 组合简洁优雅，覆盖大多数多 Agent 场景
- 与 ModelScope 生态深度集成，Qwen 系列支持最好
- 中文文档完整，针对中文场景专项优化
- 学术背景强，arXiv 论文，实验可复现

**缺点：**
- Star 数量远低于 LangChain/AutoGen，生态偏小
- 第三方工具集成有限，SaaS 覆盖少
- 社区较小，issue 响应速度慢
- API 快速演进，稳定性仍在提升中

### 3.7 适用场景

**推荐：** 多 Agent 角色协作分析 / 分布式 Agent 集群 / Multi-Agent 学术研究 / 国内模型优先 / 中文量化投研多视角分析

**不推荐：** 需要大量 SaaS 工具集成 / 社区支持优先的团队

## 4. AutoGen：微软的对话式多 Agent 框架

### 4.1 框架简介

- **GitHub：** microsoft/autogen（⭐ 38k+）
- **语言：** Python（v0.4 起支持 .NET）
- **发布：** 2023 年 10 月（微软研究院）
- **定位：** 对话驱动的多 Agent 系统，支持人类随时介入

AutoGen 的核心创新是 **ConversableAgent** 模型——每个 Agent 既能接收消息，也能发送消息，形成灵活的对话网络。

### 4.2 核心用法

```python
from autogen import AssistantAgent, UserProxyAgent

assistant = AssistantAgent(
    name="AI助手",
    llm_config={"model": "gpt-4o", "api_key": "YOUR_KEY"}
)

# UserProxyAgent 代表人类，可以执行代码
human_proxy = UserProxyAgent(
    name="人类代理",
    human_input_mode="TERMINATE",
    code_execution_config={"work_dir": "coding", "use_docker": False},
    is_termination_msg=lambda msg: "TERMINATE" in msg.get("content", "")
)

human_proxy.initiate_chat(
    assistant,
    message="帮我分析A股近30天AI板块的资金流向，并输出可视化图表"
)
```

### 4.3 群组对话（GroupChat）

```python
from autogen import GroupChat, GroupChatManager

researcher = AssistantAgent("研究员", llm_config=llm_config, system_message="你负责信息收集和事实核查")
analyst    = AssistantAgent("分析师", llm_config=llm_config, system_message="你负责量化分析和数据建模")
writer     = AssistantAgent("撰稿人", llm_config=llm_config, system_message="你负责将分析结果整理成高质量报告")
critic     = AssistantAgent("批评者", llm_config=llm_config, system_message="你负责挑出逻辑漏洞，确保结论严谨")

groupchat = GroupChat(
    agents=[researcher, analyst, writer, critic],
    messages=[],
    max_round=12,
    speaker_selection_method="auto"  # LLM 自动决定谁发言
)

manager = GroupChatManager(groupchat=groupchat, llm_config=llm_config)
human_proxy.initiate_chat(manager, message="撰写2026年量化投研AI工具深度报告")
```

### 4.4 v0.4 重大更新（2024 年底）

架构完全重写：完全异步（async/await）/ 模块化分层 / 更好的 Tracing / 支持跨语言

```python
import asyncio
from autogen_agentchat.agents import AssistantAgent
from autogen_agentchat.teams import RoundRobinGroupChat
from autogen_ext.models.openai import OpenAIChatCompletionClient

model_client = OpenAIChatCompletionClient(model="gpt-4o")
agent1 = AssistantAgent("研究员", model_client=model_client, system_message="专注信息收集")
agent2 = AssistantAgent("分析师", model_client=model_client, system_message="专注数据分析")
team = RoundRobinGroupChat([agent1, agent2], max_turns=6)

async def main():
    result = await team.run(task="分析AI Agent框架市场格局")
    print(result)

asyncio.run(main())
```

### 4.5 优缺点分析

**优点：** 社区最大（38k+）/ GroupChat 直觉友好 / Human-in-the-loop 最强 / 代码执行能力最强 / Azure OpenAI 无缝集成

**缺点：** v0.3->v0.4 大改升级成本高 / 多轮对话 token 消耗大 / GroupChat 复杂场景发言顺序混乱 / 分布式能力弱

### 4.6 适用场景

**推荐：** 代码生成与调试 / 半自动人工监督流程 / 数据分析可视化 / 开放式多专家讨论 / Azure 生态用户

**不推荐：** 需要高效固定工作流 / 生产环境高并发

## 5. CrewAI：角色驱动的 Agent 团队协作

### 5.1 框架简介

- **GitHub：** crewAIInc/crewAI（⭐ 28k+）
- **语言：** Python
- **发布：** 2024 年 1 月
- **定位：** 以「角色 + 任务」为核心的 Agent 团队框架

CrewAI 把 AI 协作类比为真实工作团队：每个成员有明确角色、职责和工具，共同完成目标。

### 5.2 三层核心抽象

| 抽象 | 含义 |
|------|------|
| **Agent** | 扮演具体角色，持有专属工具 |
| **Task** | 分配给 Agent 的具体工作，定义期望输出 |
| **Crew** | 组织 Agents 和 Tasks，定义协作流程 |

### 5.3 代码示例

```python
from crewai import Agent, Task, Crew, Process
from crewai_tools import SerperDevTool

search_tool = SerperDevTool()

researcher = Agent(
    role="市场研究专家",
    goal="收集关于AI Agent开发框架的最新市场信息和技术趋势",
    backstory="你有10年市场研究经验，擅长从海量信息中提炼关键洞察",
    tools=[search_tool],
    verbose=True,
    allow_delegation=False
)

analyst = Agent(
    role="技术分析师",
    goal="对收集到的信息进行深度技术分析，产出专业报告",
    backstory="你是资深技术架构师，专注于AI系统设计和工程实践",
    verbose=True,
    allow_delegation=True
)

research_task = Task(
    description="调研 AgentScope、AutoGen、CrewAI 的最新特性、社区活跃度和应用案例",
    expected_output="包含框架对比表格和关键发现的 Markdown 报告",
    agent=researcher
)

analysis_task = Task(
    description="从架构设计、性能、易用性三个维度深度分析，给出选型建议",
    expected_output="2000字以上技术分析报告，含选型决策树",
    agent=analyst,
    context=[research_task]
)

crew = Crew(
    agents=[researcher, analyst],
    tasks=[research_task, analysis_task],
    process=Process.sequential,
    verbose=2
)

result = crew.kickoff()
```

### 5.4 Hierarchical Process（层级模式）

```python
crew = Crew(
    agents=[researcher, analyst, writer],
    tasks=[...],
    process=Process.hierarchical,
    manager_llm=ChatOpenAI(model="gpt-4o"),
)
# Manager Agent 自动拆解任务 -> 分配给合适的子 Agent -> 汇总输出
```

### 5.5 优缺点分析

**优点：** 上手最快（30分钟跑通）/ Role+Task 直觉设计 / 社区增长迅猛 / 支持异步流式输出

**缺点：** 底层依赖 LangChain 版本兼容时有问题 / 长任务 token 消耗大 / 调试透明度弱 / 分布式能力弱

### 5.6 适用场景

**推荐：** 内容创作流水线 / 投研报告自动生成 / 快速 Multi-Agent 原型 / 市场调研竞品分析

**不推荐：** 精细控制对话轮次 / 高并发生产环境

## 6. MetaGPT：软件工程流程的 Agent 模拟

### 6.1 框架简介

- **GitHub：** geekan/MetaGPT（⭐ 46k+）
- **语言：** Python
- **发布：** 2023 年 6 月
- **定位：** 将软件工程团队协作流程映射为 Multi-Agent 系统

### 6.2 内置角色体系

| 角色 | 交付物 |
|------|--------|
| ProductManager（产品经理） | PRD（需求文档） |
| Architect（架构师） | 系统设计文档 |
| ProjectManager（项目经理） | 任务分解（WBS） |
| Engineer x N（工程师） | 代码文件 |
| QaEngineer（QA） | 测试报告 + Bug 反馈 |

### 6.3 一句话需求生成完整项目

```python
import asyncio
from metagpt.software_company import generate_repo

async def main():
    repo = await generate_repo(
        idea="开发一个A股技术指标计算库，支持MACD、KDJ、布林带等20个常用指标",
        investment=10.0,
        n_round=5
    )
    print(repo)

asyncio.run(main())
```

### 6.4 本质特点

MetaGPT 内置了软件工程 SOP，开箱即用但领域受限——输出物是规范的工程文档（PRD / 设计图 / 代码 / 测试）。

### 6.5 适用场景

**推荐：** 自动化软件项目开发 / 代码+文档同步生成 / 快速验证产品 Idea

**不推荐：** 非软件工程任务 / 对代码质量要求极高的生产交付

## 7. LangChain + LangGraph：生态最完整的 Agent 框架

### 7.1 框架简介

- **GitHub：** langchain-ai/langchain（⭐ 95k+）/ langchain-ai/langgraph（⭐ 12k+）
- **语言：** Python / JavaScript
- **定位：** LLM 应用开发瑞士军刀 + 支持复杂状态图的 Agent 执行引擎

LangGraph 用有向图来表达多 Agent 协作，最大优势是支持**循环和状态回溯**，这是其他框架难以做到的。

### 7.2 LangGraph 核心：状态图 + 条件边

```python
from langgraph.graph import StateGraph, END
from typing import TypedDict, Annotated
import operator

class ResearchState(TypedDict):
    topic: str
    research_notes: Annotated[list, operator.add]
    draft: str
    revision_count: int
    approved: bool

def research_node(state):
    return {"research_notes": [run_research(state["topic"])]}

def write_node(state):
    return {"draft": run_writer(state["research_notes"])}

def review_node(state):
    approved, _ = run_reviewer(state["draft"])
    return {"approved": approved, "revision_count": state["revision_count"] + 1}

def should_revise(state) -> str:
    if state["approved"] or state["revision_count"] >= 3:
        return "finish"
    return "revise"

graph = StateGraph(ResearchState)
graph.add_node("research", research_node)
graph.add_node("write", write_node)
graph.add_node("review", review_node)
graph.set_entry_point("research")
graph.add_edge("research", "write")
graph.add_edge("write", "review")
# 条件边：审核不通过则循环回写作节点
graph.add_conditional_edges("review", should_revise, {"revise": "write", "finish": END})
app = graph.compile()
```

### 7.3 适用场景

**推荐：** 复杂有状态 Agent 工作流（带循环/条件分支）/ 企业级 LLM 应用 / 工具集成需求丰富

**不推荐：** 轻量场景 / 非技术人员使用

def transfer_to_analyst():
    return analyst_agent

researcher_agent = Agent(
    name="研究员",

analyst_agent = Agent(
    name="分析师",

## 9. 横向对比：如何选型

### 9.1 核心能力对比

| 维度 | AgentScope | AutoGen | CrewAI | MetaGPT | LangGraph |
|------|-----------|---------|--------|---------|-----------|
| **上手难度** | 中 | 低 | 低 | 低 | 高 |
| **多 Agent 协作** | 5星 | 4星 | 4星 | 3星 | 4星 |
| **分布式支持** | 5星 | 2星 | 2星 | 2星 | 2星 |
| **代码执行** | 3星 | 5星 | 3星 | 4星 | 3星 |
| **工具集成** | 3星 | 4星 | 4星 | 3星 | 5星 |
| **可调试性** | 4星 | 3星 | 3星 | 3星 | 5星 |
| **中文生态** | 5星 | 3星 | 3星 | 3星 | 3星 |
| **社区规模** | 3星 | 5星 | 4星 | 5星 | 5星 |
| **生产可靠性** | 4星 | 3星 | 3星 | 3星 | 4星 |

### 9.2 选型决策树

```
你的核心需求是什么？
|
+-- 多 Agent 角色分工 + 国内模型 / 中文场景优先
|    --> AgentScope
|
+-- 代码生成 / 自动调试，或需要人类随时介入
|    --> AutoGen
|
+-- 快速搭建 Multi-Agent 原型，内容 / 报告类产品
|    --> CrewAI
|
+-- 任务就是软件开发，需要 PM -> 工程师 -> QA 完整流程
|    --> MetaGPT
|
+-- 复杂状态管理 + 精细控制流程 + 丰富工具集成
|    --> LangGraph
|
+-- 只是想快速验证 Idea，不想写太多代码
     --> n8n / Dify（参见 n8n 工作流专题博客）
```

### 9.3 量化投研场景选型建议

| 场景 | 推荐框架 | 核心理由 |
|------|---------|---------|
| 多视角研报（多空对辩、多来源综合） | AgentScope | 多 Agent 角色协作天然适合多分析师碰撞 |
| 策略代码生成 + 自动回测 | AutoGen | 代码执行能力最强，支持沙箱 |
| 研报调研 -> 撰写 -> 审核流水线 | CrewAI | 顺序分工清晰，上手成本低 |
| 复杂策略系统 + 精细控制 | LangGraph | 状态图覆盖循环重试和条件分支 |

## 10. 实战建议

### 先想清楚业务，再选框架

```
错误：我要用 CrewAI 做一个 Agent
正确：我需要研究员、分析师、审核员三角色协作生成研报
     -> 这是顺序分工任务 -> CrewAI 或 AgentScope 都适合
```

### 从最小可用版本开始

- 第一天：两个 Agent 对话跑通
- 第一周：完整多 Agent 工作流上线
- 第一月：根据实际问题引入分布式、记忆、监控

### 可观测性不能省

| 框架 | 推荐监控方案 |
|------|------------|
| LangGraph | LangSmith（官方，最完整） |
| AutoGen | autogen-ext Tracing 模块 |
| AgentScope | AgentScope Studio / 自定义日志 |
| CrewAI | 接入 LangSmith（底层是 LangChain） |

原则：**每个 Agent 的每一步推理都应该可追溯**，否则出问题时无从排查。

## 11. 总结

| 框架 | 一句话定位 |
|------|----------|
| AgentScope | 工业级多 Agent 协作，分布式最强，中文最友好 |
| AutoGen | 对话驱动，代码执行最强，支持人类介入 |
| CrewAI | 角色团队协作，上手最快，适合内容生产 |
| MetaGPT | 软件公司 SOP，专注代码项目自动化 |
| LangGraph | 状态图控制，灵活度最高，生态最完整 |

> 多 Agent 不是银弹。引入多 Agent 会带来复杂度、token 消耗和调试难度的成倍增加。先用单 Agent 把问题跑通，遇到真正的瓶颈（质量不够、任务太长、需要并行）时，再引入多 Agent 架构。
