## 一、工具系统架构

Claude Code 的能力完全依赖工具系统。工具调用基于 Anthropic 的 **Tool Use API**：

```
模型推理 → 生成 <tool_use> block → Harness 拦截
    → 权限检查 → PreToolUse Hook → 执行工具
    → PostToolUse Hook → 生成 <tool_result>
    → 注入上下文 → 继续推理
```

## 二、完整工具清单（45+）

### 2.1 文件操作类（核心）

| 工具 | 功能 | 关键特性 |
|------|------|---------|
| `Read` | 读取文件内容 | 支持 PDF、图片（多模态）、Jupyter Notebook；可指定行范围 |
| `Write` | 创建/覆盖文件 | 强制要求先 Read 再 Write（防误覆盖）；全量写入 |
| `Edit` | 精确字符串替换 | 基于 old_string/new_string diff；old_string 必须唯一 |
| `Glob` | 文件模式匹配 | 支持 `**/*.js` 等 glob 语法；按修改时间排序 |
| `Grep` | 代码内容搜索 | 基于 ripgrep；支持正则、文件类型过滤、上下文行 |
| `NotebookEdit` | Jupyter 单元格编辑 | 支持插入/替换/删除单元格 |

### 2.2 执行类

| 工具 | 功能 | 关键特性 |
|------|------|---------|
| `Bash` | 执行 Shell 命令 | 支持后台模式（`run_in_background`）；超时控制（最长 10 分钟）；返回 stdout/stderr |
| `TaskOutput` | 获取后台任务输出 | 异步轮询；支持 block=true/false |
| `TaskStop` | 停止后台任务 | 按 task_id 终止 |

### 2.3 网络类

| 工具 | 功能 | 关键特性 |
|------|------|---------|
| `WebFetch` | 抓取网页内容 | HTML → Markdown 转换；AI 提取指定信息；15 分钟缓存 |
| `WebSearch` | 联网搜索 | 返回结构化搜索结果；支持域名过滤 |

### 2.4 协作类（Agent 编排）

| 工具 | 功能 | 关键特性 |
|------|------|---------|
| `Agent` | 启动子 Agent | 指定 `subagent_type`；`run_in_background` 后台模式；`isolation: worktree` |
| `SendMessage` | Agent 间通信 | 向指定 Agent 发送消息，继续其上下文 |
| `TodoWrite` | 任务清单管理 | 结构化任务跟踪 |
| `TaskCreate` | 创建任务 | 带状态、依赖关系的任务管理 |
| `TaskUpdate` | 更新任务状态 | pending → in_progress → completed |
| `TaskGet` | 获取任务详情 | 读取任务完整信息 |
| `TaskList` | 列出所有任务 | 查看任务总览 |

### 2.5 IDE 集成类

| 工具 | 功能 |
|------|------|
| `mcp__ide__getDiagnostics` | 获取 VS Code 诊断信息（语言服务器错误） |
| `mcp__ide__executeCode` | 在 Jupyter Kernel 执行 Python 代码 |

### 2.6 系统类

| 工具 | 功能 |
|------|------|
| `AskUserQuestion` | 向用户提问（含选项） |
| `EnterPlanMode` | 进入 Plan Mode |
| `ExitPlanMode` | 退出 Plan Mode，请求用户批准 |
| `EnterWorktree` | 创建并进入 git worktree |
| `ExitWorktree` | 退出 worktree（keep/remove） |
| `CronCreate` | 创建定时任务 |
| `CronDelete` | 删除定时任务 |
| `CronList` | 列出所有定时任务 |

## 三、工具执行管道（单次调用完整流程）

```
┌─────────────────────────────────────────────────────────────┐
│                  工具调用完整管道                             │
│                                                             │
│  模型输出 <tool_use> block                                  │
│         ↓                                                   │
│  [1] XML/JSON 解析                                          │
│      提取: tool_name, tool_id, input_params                 │
│         ↓                                                   │
│  [2] 权限检查                                               │
│      查询: Global Settings → Project Settings → Runtime     │
│      ├── ALLOW → 继续                                       │
│      ├── DENY  → 返回 Permission denied                     │
│      └── ASK   → 弹出用户确认                               │
│         ↓                                                   │
│  [3] PreToolUse Hook                                        │
│      执行: matcher 匹配的所有脚本                            │
│      BlockingError → 阻止工具执行                           │
│         ↓                                                   │
│  [4] 工具执行                                               │
│      路由到对应执行器（文件系统/Shell/网络/Agent...）         │
│      捕获: stdout, stderr, exit_code                        │
│         ↓                                                   │
│  [5] PostToolUse Hook                                       │
│      执行: matcher 匹配的所有脚本                            │
│      可修改/记录执行结果                                     │
│         ↓                                                   │
│  [6] 生成 <tool_result> block                               │
│      包含: tool_use_id, content（文本/图片/错误）            │
│         ↓                                                   │
│  [7] 注入对话历史                                           │
│      messages.append(tool_result)                           │
│         ↓                                                   │
│  触发下一轮 claude_api() 推理                               │
└─────────────────────────────────────────────────────────────┘
```

## 四、Bash 工具的特殊模式

Bash 是使用频率最高、功能最强大的工具，有几个重要模式：

### 4.1 后台执行模式

```python
# 后台启动长时间任务
task_id = Bash(
    command="npm run build",
    run_in_background=True
)

# 主线程继续其他工作...

# 稍后获取结果（阻塞等待）
output = TaskOutput(task_id=task_id, block=True, timeout=120000)
```

### 4.2 超时控制

```python
# 默认超时：120 秒
# 最长超时：600 秒（10 分钟）
Bash(command="...", timeout=300000)  # ms
```

### 4.3 使用约束（Harness 强制执行）

```
禁止用 Bash 做的事（有专用工具替代）：
  × cat/head/tail/sed → 用 Read
  × grep/rg → 用 Grep
  × find/ls → 用 Glob
  × echo > file → 用 Write

推荐用 Bash 做的事：
  ✓ git 操作
  ✓ npm/pip/cargo 包管理
  ✓ 服务启动/停止
  ✓ 测试运行
  ✓ 构建系统
```

## 六、工具错误处理

```python
# 工具执行失败时的处理策略
if tool_result.is_error:
    # 错误信息注入上下文
    # 模型自行推理：
    # 1. 诊断错误原因
    # 2. 尝试替代方案（如用 Glob 替代失败的 Read）
    # 3. 如无法解决，向用户说明
    pass

# 常见错误类型：
# - FileNotFound → 先用 Glob 查找正确路径
# - PermissionDenied → 向用户说明需要授权
# - CommandTimeout → 建议用后台模式或分拆任务
# - UniqueMatchError（Edit）→ 提供更多上下文使 old_string 唯一
```
