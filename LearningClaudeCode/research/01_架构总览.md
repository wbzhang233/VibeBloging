# Claude Code 架构总览

> 本文档是可视化 `images/00_全景架构蓝图.html` 的技术脚本。

---

## 一、定位

Claude Code 是 Anthropic 推出的**命令行原生 Agentic AI 系统**。它不是代码补全工具，而是一个能持续运行、主动规划并执行的工程师级 AI 系统。

> 设计哲学："给 AI 一个终端，让它像工程师一样工作。"

| 维度 | 传统 AI 代码补全 | Claude Code |
|------|-----------------|-------------|
| 交互方式 | 内联补全、对话框 | CLI + 自然语言指令 |
| 执行范围 | 单文件/片段 | 全仓库级别 |
| 自主性 | 被动响应 | 主动规划 + 执行 |
| 工具调用 | 无/有限 | 完整工具链 |
| 持久性 | 单次会话 | 跨会话记忆 |
| 上下文窗口 | N/A | 200K tokens（Opus 4.6: 1M） |

## 二、四层核心架构

```
┌─────────────────────────────────────────────────────┐
│   L4  扩展层 (Extension Layer)                       │
│   MCP Servers │ Hooks │ Skills │ Custom Commands     │
├─────────────────────────────────────────────────────┤
│   L3  控制层 (Control Plane)                         │
│   权限系统 │ Plan Mode │ 安全模型 │ Cost Tracking     │
├─────────────────────────────────────────────────────┤
│   L2  上下文层 (Context System)                      │
│   CLAUDE.md │ Memory Files │ 对话历史 │ Prompt Cache │
├─────────────────────────────────────────────────────┤
│   L1  执行层 (Agentic Loop)                          │
│   Perception → Reasoning → Action → Validation      │
└─────────────────────────────────────────────────────┘
```

### L1 — 执行层（Agentic Loop）

Claude Code 的工作本质是**一个持续的感知-推理-执行循环**：

```
收集上下文信息（Perception）
    ↓
模型推理（Reasoning）
    ↓
工具调用执行（Action）
    ↓
验证结果（Validation）
    ↓
任务完成？
  ├── 是 → 返回结果给用户
  └── 否 → 带新信息回到 Perception，继续
```

**关键洞见**：大部分卡住的原因不是模型能力不足，而是上下文信息出了问题——塞了太多无关内容、或者关键信息没给到。

### L2 — 上下文层（Context System）

| 类型 | 来源 | 生命周期 |
|------|------|---------|
| 固定上下文 | CLAUDE.md（项目级约定） | 每次启动自动加载 |
| 动态上下文 | 对话历史 + 工具调用结果 | 随会话增长，70% 触发压缩 |
| 持久记忆 | Memory Files（.claude/memory/） | 跨会话保留 |
| 提示缓存 | Prompt Cache | 重复内容复用，降低 token 消耗 |

### L3 — 控制层（Control Plane）

- **Plan Mode**：探索（只读）→ 确认方案 → 执行（读写），避免偏差积累
- **权限系统**：四级层级，最小权限原则，高风险操作强制确认
- **安全模型**：高危操作拦截（rm -rf、force push、DROP TABLE 等）
- **Cost Tracking**：实时 token 消耗统计，`/cost` 命令查看

### L4 — 扩展层（Extension Layer）

| 组件 | 功能 | 触发方式 |
|------|------|---------|
| MCP Servers | 连接外部服务（数据库/API/文件系统） | JSON-RPC over stdio/SSE/HTTP |
| Hooks | 事件驱动自动化脚本 | 5 个事件点（PreToolUse 等） |
| Skills | 可复用任务模板 | 自然语言匹配 |
| Custom Commands | 自定义斜杠命令 | `/command-name` |

## 三、核心组件清单

### Harness（运行时框架）

Harness 是 Claude Code 的"骨架"，7 大职责模块：

| 模块 | 职责 |
|------|------|
| 消息管理 | 维护对话历史、上下文窗口边界 |
| 工具调度 | 拦截 tool_call，路由到对应执行器 |
| 权限检查 | 验证操作合法性，拦截高危命令 |
| Hooks 触发 | 在 5 个事件点执行自定义脚本 |
| 子代理管理 | 生成/监控/回收子 Agent 进程 |
| 错误恢复 | retry/fallback 策略 |
| 流式输出 | 实时推理展示 + 会话持久化 |

## 四、启动序列

```
1. claude 命令启动
2. 查找并加载 CLAUDE.md（当前目录 → 父目录 → 全局）
3. 加载 Memory Files（.claude/memory/）
4. 加载 settings.json（全局 → 项目级）
5. 初始化 MCP Server 连接
6. 组装 System Prompt
7. 进入 QueryLoop
```
