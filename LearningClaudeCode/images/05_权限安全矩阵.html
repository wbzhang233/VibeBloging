四级权限层级（同心矩形）

1
Global Settings
~/.claude/settings.json

全局白名单/黑名单，对所有项目和会话生效

2
Project Settings
.claude/settings.json

项目级权限，覆盖全局设置，可提交到 git

3
Runtime Authorization
会话内临时授权

用户在会话中批准特定操作，会话结束失效

4
One-time Approval
单次允许
仅此一次，不记录到任何配置文件

⚠ 高危操作拦截清单

⛔
rm -rf / · rm -rf * · rm -rf ./强制拒绝
⛔
curl * | bash强制拒绝
⚡
git push --force · git push -f强制确认
⚡
git reset --hard · git checkout .强制确认
⚡
DROP TABLE · TRUNCATE *强制确认
⚡
.github/workflows/ 修改强制确认
⚡
Slack / Email / GitHub 发消息强制确认
⚡
~/.ssh/id_rsa 读取并外传强制确认
⚡
sudo · chmod 777强制确认
~/.claude/settings.json 
// 全局权限配置
{
  "permissions": {
    "allow": [
      "Bash(git *)",
      "Bash(npm *)",
      "Bash(python *)",
      "Read(*)",
      "Write(*)",
      "Edit(*)"
    ],
    "deny": [
      "Bash(rm -rf *)",
      "Bash(git push --force *)",
      "Bash(curl * | bash)",
      "Bash(sudo *)"
    ]
  }
}

Plan Mode · 两阶段安全隔离

探索阶段
只读区
✓ Read / Glob / Grep
✓ WebFetch / WebSearch
✓ Agent(Explore/Plan)
✗ Write / Edit
✗ Bash（含执行）
执行阶段
全功能区
✓ 全部工具
✓ Write / Edit / Bash
✓ Agent(general)
须用户批准后进入

操作风险矩阵

工具 / 操作 可逆性 影响范围 风险级别 默认处理
Read / Glob / Grep极高只读低风险自动允许
WebFetch / WebSearch高只读低风险自动允许
Write / Edit（新文件）中本地文件中风险会话授权
Bash（读取命令）高只读低-中规则匹配
Bash（写入/安装）中本地系统中风险规则匹配
git push --force低远程仓库高风险强制确认
外部消息发送低外部系统高风险强制确认
DROP TABLE / rm -rf不可逆数据库/文件极端风险拒绝/确认
