# 三款 AI Agent 横向对比：Claude Code、OpenClaw、Hermes Agent

> 2025 年下半年以来，AI Agent 领域经历了一次显著的产品分化：
> 官方 Claude Code 持续拓宽边界、OpenClaw（前身 Clawdbot）以全天候个人 AI 助理赛道崛起、Hermes Agent 凭借持久记忆引爆社区。
> 三者并非替代关系，但选错了会让你事倍功半。

---

## 背景：三个产品从哪里来

**Claude Code**（官方版）是 Anthropic 于 2024 年推出的终端原生 AI 编码助手，起初定位垂直编码场景，后持续迭代，向通用智能体方向大幅扩展，新增了 Computer Use 桌面控制、Agent Teams 多代理协同、Dispatch 移动端远程控制等核心能力，逐步演进为"以编码能力为核心护城河的全场景端侧智能体"。

**OpenClaw**（前身为 Clawdbot）是由独立开发者 Peter Steinberger 主导创建的完全自主开源 Agent 项目，与 Claude Code 源码无任何直接关联。最初以 Clawdbot 名称在社区发布，短期内 Discord 服务器迅速扩至 5000+ 成员，吸引了众多 AI 领域创始人和 KOL 关注，后更名为 OpenClaw。它以"全天候个人 AI 助理"为核心定位，通过 Markdown 记忆文件系统、自主技能生成、IM 网关和心跳机制，实现跨平台持久化运行，支持 WhatsApp、Discord、Telegram 等多 IM 平台接入，可部署在 Mac mini 或低成本 VPS 上 7×24 小时运行。Hermes 官方文档提供了从 OpenClaw 迁移到 Hermes 的指引，可见两者在使用人群上有较大重叠。

**Hermes Agent** 是 Nous Research 于 2025 年 2 月开源的通用持久化自主智能体，以单 Agent 闭环架构、四层分层记忆、技能自动蒸馏为核心差异点，主打"越用越懂你"的长期助理场景，完全 MIT 开源，含模型权重与训练方法。

## 一、核心定位：三种不同的产品基因

| 维度 | Claude Code（官方） | OpenClaw | Hermes Agent |
|------|--------------------|--------------------|--------------|
| **出品方** | Anthropic 官方 | 独立开发者 Peter Steinberger 主导，Clawdbot 更名而来，MIT 完全开源 | Nous Research 完全开源 |
| **核心定位** | 专业级端侧执行智能体，编码为核心护城河，向全场景扩展 | 全天候个人 AI 助理，以持久记忆、IM 多平台接入与心跳主动感知为核心 | 通用持久化自主智能体，核心是"越用越懂你" |
| **产品哲学** | Unix 哲学 + 精准执行优先，安全可控 | 单 Agent 闭环，个人助理感，Markdown 驱动记忆，心跳机制实现主动感知 | 单 Agent 闭环，自主进化，极简自托管 |
| **典型用户** | 专业开发者、工程师、需要 Anthropic 模型顶级能力的用户 | 需要 7×24 在线个人助理、多 IM 平台统一管理、低成本自托管的个人用户 | 需要长期常驻、跨平台、自动化程度高的个人用户 |

**核心区别一句话**：
- Claude Code 官方 = **最强编码工具**，向通用延伸
- OpenClaw = **全天候个人 AI 助理**，心跳驱动主动感知，IM 多平台接入
- Hermes Agent = **不一样的物种**，优先解决长期记忆和自主进化问题

## 二、架构设计：多 Agent 编排 vs 单 Agent 持久循环

这是理解三者本质差异最重要的维度。

### Claude Code 官方：多 Agent 编排架构

主代理（Coordinator）负责任务拆解，多个子代理并行执行，每个子代理拥有独立上下文窗口和 git worktree 隔离环境，结果汇总后交付。这套架构适合**大型项目拆分、多模块并行开发**等复杂工程任务，稳定性和隔离性强。

网关层仅负责消息交付，技能创建、记忆写入、定时任务均为独立模块，相互解耦。

### OpenClaw：独立自研，单 Agent 闭环 + 心跳驱动

OpenClaw（前身 Clawdbot）并非 Claude Code 架构的延伸，而是一套**完全独立开发**的单 Agent 持久循环架构，设计理念与 Hermes Agent 更为接近：

```
输入 → 推理 → 工具使用 → 记忆更新 → 技能沉淀 → 结果输出
```

**心跳机制（Heartbeat）** 是 OpenClaw 的标志性设计：系统每小时向 Agent 发送一次内部 prompt，Agent 自主判断当前是否有需要主动处理的事项。若无需操作则返回 `heartbeat OK` 信号，消耗极少 Token；若有待处理任务则主动触发执行，真正实现主动感知与持续在线。

**IM 网关**原生支持 WhatsApp、Discord、Telegram 等多平台，会话绑定用户 ID，跨平台上下文无缝续接。网关与主循环深度融合，消息路由、定时触发、记忆写入在同一闭环内完成。

### Hermes Agent：单 Agent 持久循环

没有编排层，没有多 Agent 集群。所有任务走同一条循环：

```
输入 → 推理 → 工具使用 → 记忆更新 → 技能蒸馏 → 结果输出
```

网关深度融入主循环：消息路由、技能创建、定时触发、记忆写入，全部在同一个闭环内完成。会话绑定用户 ID，不绑定平台——Telegram 开启的对话，切到终端或飞书，上下文无缝续接。

| 架构对比 | Claude Code 官方 | OpenClaw | Hermes Agent |
|---------|-----------------|----------|--------------|
| **核心范式** | 多 Agent 编排 | 单 Agent 持久循环（心跳驱动） | 单 Agent 持久循环 |
| **网关设计** | 与执行解耦 | 原生 IM 网关，深度融入主循环 | 深度融入主循环 |
| **会话绑定** | 终端/项目 | 用户 ID（原生） | 用户 ID（原生） |
| **技术栈** | TypeScript + Bun + Ink | Python，支持主流 LLM API 与本地模型 | Python + SQLite + Ollama |
| **开源状态** | 官方核心闭源 | MIT 完全开源，独立自研，非 Claude Code 衍生 | MIT 完全开源 |

## 三、记忆系统：三种截然不同的设计取向

记忆系统的差异，直接决定了长期使用体验。

### Claude Code 官方：四层规则化记忆，人工主导

1. **项目记忆（CLAUDE.md）**：手动维护，当前项目全局规则与上下文
2. **用户记忆（~/.claude/claude.md）**：个人操作偏好与长期规则
3. **会话记忆**：单次会话上下文，`/resume` 跨会话恢复
4. **团队共享记忆（Agent Teams）**：多代理协作时共享的任务上下文

本质是**人工定义**：你写什么，它记什么。官方版默认全量加载记忆文件，无按需检索，上下文过长时自动压缩。

### OpenClaw：Markdown 文件驱动，结构化分层记忆

OpenClaw 采用 Markdown 文件管理记忆，分层设计：

1. **Agent 上下文**：会话内临时状态，会话结束后重置
2. **工作记忆（.md 文件）**：跨会话持久化，每次会话结束后自动摘要写入，可快速定位上次进展
3. **Memory.md**：用户偏好、知识库与长期设置，每次会话开始时加载
4. **SQLite 混合检索**：将 Markdown 切分为约 400 Token 的小块，70% 语义匹配 + 30% 关键词精确匹配，按需检索

记忆更新触发：任务完成或会话结束时，Agent 自主判断哪些信息值得写入，无需用户干预，与 Hermes 的 periodic nudge 机制类似。

### Hermes Agent：四层分层记忆，Agent 自主蒸馏

1. **提示记忆**：`MEMORY.md` + `USER.md`，常驻上下文，严格限制 3,575 字符
2. **会话检索**：SQLite + FTS5 索引，仅在相关时检索，经 LLM 摘要后使用
3. **技能程序性记忆**：存"怎么做"，按需懒加载，大量技能也不推高 Token 成本
4. **Honcho 用户建模**：跨会话被动追踪偏好、风格、知识背景

关键机制：**periodic nudge（定期提醒）**。会话中，系统定期向 Agent 发送内部系统级提示，让它评估哪些内容值得写进记忆文件，全程无需用户干预。

| 记忆对比 | Claude Code 官方 | OpenClaw | Hermes Agent |
|---------|-----------------|----------|--------------|
| **更新触发** | 用户手动 + 会话后自动整合 | 任务完成后自动写入，Agent 自主判断 | 系统自动，Agent 自主判断 |
| **检索策略** | 默认全量加载 | SQLite 混合检索（语义 + 关键词） | 按需检索 + LLM 摘要 |
| **跨会话一致性** | 依赖 CLAUDE.md 手动维护 | 自动持续积累，工作记忆跨会话不丢失 | 自动持续积累，不丢失 |
| **核心价值取向** | 统一执行规范 | 个人助理感，记住你的偏好与进展 | 越用越懂你 |

## 四、技能系统：插件 vs 自我进化

### Claude Code 官方：手动编写 SKILL.md

技能由用户编写 `SKILL.md` 文件定义，固化特定场景的工作流。支持懒加载，内置 45+ 工具（Bash、Git、文件操作、浏览器等），依赖 MCP 服务器扩展更多能力。技能本质是**静态插件**：你定义它能做什么，它就只做这些。

### OpenClaw：自动蒸馏，与 Hermes 同路线

OpenClaw 的技能系统同样支持自动生成：任务完成后，Agent 评估流程有效性，自动将可复用方法提炼为 skill 文件，无需人工编写。下次遇到类似场景直接调用，使用频次越高，适配性越强。技能本质是**自主沉淀的程序性记忆**。

### Hermes Agent：自动蒸馏，无需人工编写

完成任务后，Hermes 自动评估处理流程是否值得保留，提炼有效方法写入 `~/.hermes/skills/`。下次遇到类似任务，直接调用已沉淀的工作流。技能本质是**自我进化的程序性记忆**：Agent 自己学会的，不需要你告诉它。

| 技能对比 | Claude Code 官方 | OpenClaw | Hermes Agent |
|---------|-----------------|----------|--------------|
| **生成方式** | 完全手动编写 | Agent 自主生成，完成任务后自动评估沉淀 | Agent 完全自主生成 |
| **更新迭代** | 人工维护 | 基于执行日志自主迭代 | 基于执行日志自主迭代 |
| **加载策略** | 按需懒加载 | 按需加载，节省 Token 开销 | 默认只加载名称摘要，极致省 Token |
| **泛化能力** | 仅匹配预设场景 | 适配同类场景不同需求 | 适配同类场景不同需求 |

## 七、选型建议

### 优先选 Claude Code 官方的场景
- 核心需求是**专业编码开发**、大型项目管理、Code Review、CI/CD
- 需要 Anthropic Claude 模型的顶级代码理解与生成能力
- 使用 **Computer Use** 做桌面自动化操作
- 需要企业级权限管控与成本跟踪
- 参与 **Agent Teams** 多人协作的工程场景

### 优先选 OpenClaw 的场景
- 需要一个 **7×24 在线、IM 多平台接入（特别是 WhatsApp）** 的个人 AI 助理
- 希望 Agent **主动感知**任务状态，而非纯被动等待指令（心跳机制）
- 追求**极简部署**，5 美元 VPS 或闲置 Mac mini 即可跑起来
- 已有 **OpenClaw/Clawdbot 社区**生态与使用习惯
- 需要 Agent 自主沉淀工作流技能，又不想完全迁移到 Hermes

### 优先选 Hermes Agent 的场景
- 需要一个 **24h 在线、跨平台、重启不丢记忆**的个人 AI 助理
- 有大量**重复性、长周期任务**，希望 Agent 自主学习并沉淀工作流
- 追求**极致成本控制**，愿意自托管或使用本地模型
- 需要**多工具协同**（Gmail、日历、Todoist、Obsidian 等个人工作流）
- 希望 Agent 之间可以**无人干预地互相协作**

### 最优实践：三者协同

三个工具并非非此即彼，最优解往往是组合：

```
Hermes Agent（全局中枢）
    ├── 长期任务调度与跟踪
    ├── 跨平台消息处理
    ├── 个性化工作流深度沉淀
    └── 调度 → Claude Code（专业执行单元）
                  ├── 编码开发任务
                  ├── Code Review
                  └── CI/CD 与工程化落地

OpenClaw（个人助理方案）
    └── 当需要心跳主动感知、WhatsApp 接入，或
        作为 Hermes 的轻量替代方案时
```

有用户已经在实践这种协同：用 Hermes 把 Gmail、日历、Todoist 信息汇总分析，把编码任务分派给 Claude Code，持续跟踪进度并回写到 Obsidian vault。全程无人工介入。

## 八、总结

| | Claude Code 官方 | OpenClaw | Hermes Agent |
|---|---|---|---|
| **一句话定位** | 专业编码的顶级执行工具 | 心跳驱动的全天候个人 AI 助理 | 自主进化的个人 AI 管家 |
| **最大优势** | 编码能力 + 工程生态 | IM 全平台接入 + 心跳主动感知 + 极低部署成本 | 持久记忆 + 自主学习 |
| **核心短板** | 模型绑定 + 非编码场景弱 | 编码能力弱，生态相比 Claude Code 尚浅 | 编码能力一般，新兴生态 |
| **适合人群** | 专业开发者 | 需要 7×24 在线个人助理、多 IM 管理的用户 | 需要长期 AI 助理的用户 |

AI Agent 正在从"用完即走的工具"走向"持续成长的协作伙伴"。Claude Code 代表了前者的极致；Hermes Agent 与 OpenClaw 都在探索后者——Hermes 以深度记忆蒸馏和自主进化为核心，OpenClaw 以心跳主动感知和多 IM 平台接入为亮点，两者路线相近却各有侧重。

选哪个，取决于你更在意今天的执行效率，还是明天的长期积累——以及你需要的是主动找你的 AI 助理，还是需要你去找的专业工具。
