Hermes Agent
记忆进化架构

Nous Research 打造的自进化智能体。四层记忆分级调度，Skill 自动蒸馏沉淀，Periodic Nudge 周期性唤醒，Honcho 用户建模——Agent 越用越强。

任务执行 → 效果评估 → Skill 蒸馏 → 知识沉淀 → 下次调用 ↺ 越用越强
四层记忆层次详解
L1 · PROMPT MEMORY
系统提示记忆
3575 chars · 恒定注入
每次对话初始化时注入 system prompt，包含 Agent 人格定义、核心规则和工具调用指南。这是最热、最快的记忆路径——始终在上下文窗口中，零检索延迟。
人格定义 工具规则 0延迟
L2 · SESSION SEARCH
会话语义检索
SQLite FTS5 + 向量
跨会话持久化对话历史，支持全文检索（FTS5）与语义向量检索双路并行。每次对话前根据当前输入检索最相关的历史片段注入上下文，实现长期记忆。
跨会话 双路检索 持久化
L3 · SKILLS LIBRARY
技能程序库
YAML · 自动蒸馏
Agent 完成任务后自动评估执行过程，将可复用的操作序列蒸馏为 YAML Skill 文件。下次遇到相似任务时，Agent 优先调用已有 Skill，跳过重新推理阶段。
自动蒸馏 YAML格式 越用越强
L4 · HONCHO
Honcho 用户建模
动态偏好推断
独立的用户建模层，持续追踪用户偏好、习惯和反馈模式，构建动态用户画像。Agent 根据用户模型调整响应风格、主动程度和专业深度，实现真正的个性化适配。
用户画像 偏好追踪 个性化
Skill 自动蒸馏流水线
Periodic Nudge 周期唤醒机制
⏱️
定频触发
每隔 N 轮对话（nudge_frequency 可配），系统自动注入特殊提示，刷新 Agent 的记忆锚点与行为规范，防止长对话中的规则漂移
🧭
记忆巩固
Nudge 内容包含用户建模摘要、重要规则重申、近期 Skill 更新等，相当于 Agent 的"工作记忆刷新"，确保长期运行时的行为一致性
🔄
无缝透明
Nudge 对用户完全透明，不打断对话流程。Agent 在内部悄然更新状态，像人类一样在长时间对话中通过自我提醒保持专注与一致
👤 Honcho 用户建模层
Honcho 是 Hermes Agent 的第四层记忆，也是它与竞品最大的差异化所在。不同于通用 Agent 对所有用户一视同仁，Hermes 持续观察用户的交互模式、反馈信号和偏好习惯，动态构建用户画像。

随着使用时间增长，Agent 越来越"了解"这个用户——知道他喜欢简洁还是详细的回答、擅长哪些领域、什么话题需要谨慎。这种个性化适应能力，让 Hermes 从工具进化为真正的私人助理。
