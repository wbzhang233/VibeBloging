# Claude Code 与 Hermes Agent 深度对比报告
本报告基于 Claude Code 官方最新稳定版（v2.1.88，2026年3月）、社区开源修复版（cc-haha）、官方源码还原库，以及 Hermes Agent 官方开源v0.7.0版本（2026年4月）、官方文档与社区实测数据，从九大核心维度做全维度对标，所有内容均采用表格化呈现，清晰直观呈现二者的核心异同与选型参考。

## 二、核心架构设计与底层范式对比
| 对比维度       | Claude Code                                                              | Hermes Agent                                                                 |
| -------------- | ------------------------------------------------------------------------ | ---------------------------------------------------------------------------- |
| 核心架构范式   | 多Agent编排架构，基于Coordinator协调器实现主代理+子代理的分工协同执行，支持Agent Teams团队模式        | 单Agent持久循环架构，无独立编排层，无多Agent集群设计，所有任务均通过统一核心循环完成                         |
| 核心执行链路   | 用户输入→主代理任务拆解→子代理并行执行→结果汇总校验→权限审批→最终交付                      | 统一单循环链路：输入→推理→工具使用→记忆更新→技能蒸馏→结果输出，全程无额外调度环节                         |
| 网关与路由设计 | 官方版网关仅负责消息交付，技能创建、记忆写入、定时任务均为独立机制，与网关解耦；社区版扩展了Channel网关系统 | 网关深度融入主循环，消息传递、会话路由、技能创建、定时触发、记忆写入均在同一闭环内完成，是系统核心模块 |
| 会话绑定规则   | 官方版绑定终端/当前项目，跨终端、跨平台上下文连续性较弱；社区版支持绑定用户ID，实现多平台会话续接                          | 原生绑定用户ID，不绑定具体平台/设备，Telegram、终端、飞书、Discord等多入口可无缝续接会话，无上下文丢失       |
| 核心技术栈     | TypeScript + Bun + React + Ink + Anthropic SDK，社区版扩展了多模型SDK适配                           | Python + SQLite + Ollama，原生兼容各类LLM接入与MCP服务器协议                                    |
| 开源合规属性   | 官方核心代码闭源，社区版基于泄露源码修复适配，非官方全量开源；工具链与MCP规范逐步开放                             | 完全开源（MIT协议），含模型权重、训练数据集、训练方法与系统全量源码，无闭源核心模块                      |

## 三、记忆系统核心能力对比
| 对比维度         | Claude Code                                                                                                                           | Hermes Agent                                                                                                                                                                                                                                      |
| ---------------- | ------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 整体记忆架构     | 四层规则化+自主整合记忆架构，人工定义与后台自动整合结合                                                                                                    | 四层分层式记忆架构，以Agent自主蒸馏、自主管理为核心                                                                                                                                                                                                           |
| 记忆更新触发机制 | 官方版支持用户手动编写/更新记忆文件，搭配auto-dream后台状态机，会话结束后异步完成记忆整合；仅支持手动+会话后自动两种触发模式                                                                                 | 系统自动定期触发内部系统级提示，Agent自主评估会话内容，判断哪些信息值得留存、哪些流程值得沉淀，全程无需用户干预，会话中即可完成记忆更新                                                                                                                                                                                   |
| 记忆分层设计     | 1. 项目记忆（CLAUDE.md）：当前项目全局规则与上下文<br>2. 用户记忆（全局claude.md）：个人操作偏好与长期规则<br>3. 会话记忆：单次会话上下文，支持跨会话检索<br>4. 团队共享记忆：Agent Teams模式下多代理共享的任务上下文与协作规则 | 1. 提示记忆：常驻上下文，严格限制3575个字符，仅保留核心长期信息<br>2. 会话检索：SQLite+FTS5索引，仅在任务相关时主动检索，经LLM摘要后仅保留相关内容，不占用常驻上下文<br>3. 技能程序性记忆：存储可复用工作流，按需懒加载<br>4. Honcho用户建模：跨会话被动追踪用户偏好、沟通风格与知识背景 |
| 检索与复用逻辑   | 官方版默认全量加载记忆文件，无精细化按需检索机制，易造成Token浪费；社区版优化了按需检索与懒加载能力                                                                                     | 仅在判断当前任务与历史会话相关时，才主动检索对应内容；技能完整内容仅在任务确需使用时加载，极致控制Token开销，即便积累上百个技能也不会推高调用成本                                                                                                                                                                     |
| 核心设计目标     | 统一操作规范与执行标准，对齐用户的使用习惯，保障复杂任务的执行一致性                                                                                                      | 实现长期记忆不丢失、不混乱，同时不推高Token成本，让Agent越用越适配用户，彻底解决传统Agent跨会话失忆、记忆冗余的行业痛点                                                                                                                                                                              |

## 总结
Claude Code 与 Hermes Agent 并非非此即彼的竞品关系，而是定位高度互补、能力可深度协同的两类AI智能体产品。

Claude Code 是Anthropic官方打造的、从编码垂类进化而来的**端侧执行型智能体**，核心优势集中在代码理解与工程化落地、桌面级全场景操作、精准权限管控与多Agent团队协同，是深度嵌入开发者与办公人群工作流的顶级执行副手，在专业场景下的稳定性、精准度与安全性无可替代。

而 Hermes Agent 是Nous Research推出的**通用型持久化自主进化智能体**，核心突破在于单Agent闭环架构、自主蒸馏的分层记忆系统、极低的自托管成本与跨平台常驻能力，更适合作为7×24小时在线的个人任务管家与工作流中枢，实现长期任务跟踪、跨平台协同与自主能力进化。

对于用户而言，二者的最优解并非二选一，而是形成协同工作流：**以 Hermes Agent 作为全局中枢，负责长期任务调度、跨平台消息处理、个性化工作流沉淀与全局统筹；以 Claude Code 作为专业执行单元，专注编码开发、工程化落地、端侧系统操作等专业技术任务**，最大化发挥二者的核心优势，搭建完整的个人AI提效体系。
